# 方案一：Express + React 全栈博客平台

## 项目介绍

### 项目是什么
这是一个功能完整的博客平台，支持文章发布、分类管理、评论互动功能。采用前后端分离架构，前端使用 React + TypeScript，后端使用 Express + TypeScript，数据库使用 MongoDB。

### 解决什么问题
- 为技术博客提供一个专业的发布平台
- 展示 TypeScript 全栈开发能力
- 学习 RESTful API 设计规范

### 核心技术亮点
- **TypeScript 严格类型检查**：前后端共享类型定义
- **Express 中间件机制**：统一的错误处理和日志记录
- **React Hooks 状态管理**：函数式组件最佳实践
- **MongoDB 文档型数据库**：灵活的 schema 设计

### 适合什么场景
- 个人技术博客
- 团队内部文档系统
- 简单的 CMS 内容管理系统

```typescript
/**
 * MongoDB 数据库配置文件
 * 
 * 为什么需要单独的配置文件？
 * 1. 集中管理数据库连接参数
 * 2. 方便切换开发/生产环境
 * 3. 保护敏感信息（密码等）
 */

import mongoose, { ConnectOptions } from 'mongoose';

// 从环境变量获取数据库配置，没有则使用默认值
// process.env 是 Node.js 读取环境变量的标准方式
const MONGODB_URI = process.env.MONGODB_URI || 'mongodb://localhost:27017/blog';
const MONGODB_USER = process.env.MONGODB_USER || '';
const MONGODB_PASSWORD = process.env.MONGODB_PASSWORD || '';

/**
 * 构建完整的 MongoDB 连接字符串
 * 支持两种格式：
 * 1. 标准格式: mongodb://host:port/database
 * 2. 认证格式: mongodb://user:password@host:port/database
 */
const getConnectionString = (): string => {
  // 如果有认证信息，构建带认证的连接字符串
  if (MONGODB_USER && MONGODB_PASSWORD) {
    return `mongodb://${MONGODB_USER}:${MONGODB_PASSWORD}@localhost:27017/blog?authSource=admin`;
  }
  return MONGODB_URI;
};

/**
 * 数据库连接选项
 * 这些选项影响连接行为和性能
 */
const connectionOptions: ConnectOptions = {
  // 最大连接池大小，默认100
  // 连接池是一组预先建立的数据库连接，避免频繁创建/销毁连接的开销
  maxPoolSize: 10,
  
  // 连接超时时间（毫秒）
  // 如果超过这个时间还没连接成功，就放弃
  serverSelectionTimeoutMS: 5000,
  
  // socket 超时时间（毫秒）
  // 如果一个操作超过这个时间没响应，就认为超时
  socketTimeoutMS: 45000,
};

/**
 * 连接数据库的核心函数
 * 使用 async/await 处理异步操作
 */
export const connectDatabase = async (): Promise<void> => {
  try {
    console.log('🔄 正在连接 MongoDB...');
    
    // mongoose.connect 返回一个 Promise
    // 使用 await 等待连接完成
    await mongoose.connect(getConnectionString(), connectionOptions);
    
    console.log('✅ MongoDB 连接成功！');
    console.log(`📊 数据库地址: ${getConnectionString().split('@')[1] || 'localhost'}`);
    
  } catch (error) {
    // 连接失败时打印错误并退出程序
    // 在生产环境中，可能需要更优雅的错误处理
    console.error('❌ MongoDB 连接失败:', error);
    process.exit(1); // 退出进程，退出码 1 表示异常退出
  }
};

/**
 * 断开数据库连接
 * 通常在应用关闭时调用
 */
export const disconnectDatabase = async (): Promise<void> => {
  try {
    await mongoose.disconnect();
    console.log('📤 MongoDB 连接已断开');
  } catch (error) {
    console.error('断开连接失败:', error);
  }
};

/**
 * 监听数据库连接事件
 * 用于监控连接状态和处理重连逻辑
 */

// 当连接打开时触发
mongoose.connection.on('connected', () => {
  console.log('📡 MongoDB 连接已建立');
});

// 连接断开时触发（可能是网络问题或手动断开）
mongoose.connection.on('disconnected', () => {
  console.log('📡 MongoDB 连接已断开');
});

// 连接出错时触发
mongoose.connection.on('error', (err) => {
  console.error('📡 MongoDB 连接错误:', err);
});

// mongoose.connection 是 Mongoose 内置的事件发射器
// 可以监听各种数据库事件进行监控
```

```typescript
/**
 * 用户数据模型
 * 
 * Mongoose 模型是对 MongoDB 集合的操作封装
 * 定义模型时指定 schema，模型会自动创建对应的集合
 */

import mongoose, { Schema, Document } from 'mongoose';

// 引入类型定义（用于 TypeScript 类型检查）
import { IUser } from '../types';

// 合并 Document 和 IUser，得到完整的用户文档类型
// Document 是 Mongoose 提供的基类，包含 _id 等默认属性
export interface UserDocument extends IUser, Document {
  // 这里可以添加文档方法（实例方法）
  // 例如: comparePassword(password: string): Promise<boolean>
}

// 创建用户 Schema
// Schema 是 MongoDB 文档结构的定义
const userSchema = new Schema<UserDocument>(
  {
    // 用户名字段
    username: {
      type: String,              // 数据类型
      required: [true, '用户名不能为空'],  // 必填验证
      unique: true,              // 唯一索引
      trim: true,                // 自动去除首尾空格
      minlength: [3, '用户名至少3个字符'],  // 最小长度
      maxlength: [20, '用户名最多20个字符'], // 最大长度
    },
    
    // 邮箱字段
    email: {
      type: String,
      required: [true, '邮箱不能为空'],
      unique: true,              // 唯一索引
      lowercase: true,           // 自动转为小写
      match: [                    // 正则验证
        /^\w+([.-]?\w+)*@\w+([.-]?\w+)*(\.\w{2,3})+$/,
        '请输入有效的邮箱地址',
      ],
    },
    
    // 密码字段
    password: {
      type: String,
      required: [true, '密码不能为空'],
      minlength: [6, '密码至少6个字符'],
      select: false,             // 查询时默认不返回密码
    },
    
    // 头像字段
    avatar: {
      type: String,
      default: 'https://api.dicebear.com/7.x/avataaars/svg?seed=default',
    },
    
    // 个人简介
    bio: {
      type: String,
      maxlength: [200, '简介最多200个字符'],
    },
  },
  {
    // Schema 配置选项
    timestamps: true,             // 自动添加 createdAt 和 updatedAt
    
    // toJSON 配置，控制 JSON 序列化行为
    toJSON: {
      transform: (_doc, ret) => {
        // 删除敏感字段
        delete ret.password;
        delete ret.__v;
        return ret;
      },
    },
  }
);

/**
 * 索引设计
 * 索引可以加速查询，但会增加写入成本
 */

// 复合索引，同时查询用户名和邮箱时效率更高
userSchema.index({ username: 1, email: 1 });

// 文本索引，用于搜索用户
userSchema.index({ username: 'text', bio: 'text' });

/**
 * 实例方法
 * 可以在模型实例上调用
 */

/**
 * 密码比对方法
 * @param candidatePassword 用户输入的密码
 * @returns 密码是否匹配
 */
userSchema.methods.comparePassword = async function (
  this: UserDocument,
  candidatePassword: string
): Promise<boolean> {
  // 使用 bcrypt 比对密码
  // bcrypt.compare 会自动处理盐值
  const bcrypt = await import('bcryptjs');
  return bcrypt.compare(candidatePassword, this.password);
};

/**
 * 静态方法
 * 可以在模型上直接调用
 */

/**
 * 根据邮箱查找用户（包含密码字段）
 * @param email 用户邮箱
 */
userSchema.statics.findByEmailWithPassword = async function (
  email: string
): Promise<UserDocument | null> {
  return this.findOne({ email }).select('+password');
};

/**
 * 创建用户模型
 * 第一个参数是模型名，第二个是 Schema，第三个是集合名（可选）
 */
export const User = mongoose.model<UserDocument>('User', userSchema, 'users');

export default User;
```

**src/models/Post.ts**
```typescript
/**
 * 文章数据模型
 * 展示更复杂的数据关系和验证
 */

import mongoose, { Schema, Document, Model } from 'mongoose';
import { IPost, PostStatus } from '../types';

// 文章文档类型
export interface PostDocument extends IPost, Document {
  // 可添加文章特有的实例方法
}

// 获取关联的作者信息（虚拟字段）
// 虚拟字段不存储在数据库中，但在转换为 JSON 时可以包含
const postSchema = new Schema<PostDocument>(
  {
    // 标题
    title: {
      type: String,
      required: [true, '文章标题不能为空'],
      trim: true,
      maxlength: [200, '标题最多200个字符'],
    },
    
    // 内容（Markdown）
    content: {
      type: String,
      required: [true, '文章内容不能为空'],
      maxlength: [50000, '内容最多50000个字符'],
    },
    
    // 摘要（自动从内容提取或手动设置）
    excerpt: {
      type: String,
      maxlength: [500, '摘要最多500个字符'],
    },
    
    // URL 友好的标识符
    // 例如: "my-first-post" 而不是 "64f8a1234567890abcdef"
    slug: {
      type: String,
      required: [true, 'URL 别名不能为空'],
      unique: true,
      lowercase: true,
      trim: true,
      // slug 应该只包含字母、数字、连字符
      match: [/^[a-z0-9]+(?:-[a-z0-9]+)*$/, 'URL 别名格式不正确'],
    },
    
    // 封面图片
    coverImage: {
      type: String,
      default: null,
    },
    
    // 作者（引用 User 模型）
    // ref 指定关联的模型名
    author: {
      type: Schema.Types.ObjectId,
      ref: 'User',               // 关联到 User 模型
      required: true,
      index: true,               // 添加索引加速查询
    },
    
    // 分类
    category: {
      type: Schema.Types.ObjectId,
      ref: 'Category',
      required: [true, '文章分类不能为空'],
      index: true,
    },
    
    // 标签
    tags: {
      type: [String],           // 字符串数组
      default: [],
      // 验证每个标签
      validate: {
        validator: function (v: string[]) {
          return v.length <= 10; // 最多10个标签
        },
        message: '标签最多10个',
      },
    },
    
    // 发布状态
    status: {
      type: String,
      enum: Object.values(PostStatus),  // 枚举值
      default: PostStatus.Draft,
      index: true,
    },
    
    // 阅读数
    viewCount: {
      type: Number,
      default: 0,
      min: 0,
    },
    
    // 点赞数
    likeCount: {
      type: Number,
      default: 0,
      min: 0,
    },
    
    // 评论数
    commentCount: {
      type: Number,
      default: 0,
      min: 0,
    },
    
    // 发布时间（仅已发布的文章有）
    publishedAt: {
      type: Date,
      default: null,
    },
  },
  {
    timestamps: true,
    toJSON: {
      transform: (_doc, ret) => {
        ret.id = ret._id;
        delete ret._id;
        delete ret.__v;
        return ret;
      },
    },
  }
);

/**
 * 索引设计
 */

// 复合索引：按状态和发布时间排序
// 常见查询：获取已发布文章，按最新排序
postSchema.index({ status: 1, publishedAt: -1 });

// 复合索引：按作者和状态查询
postSchema.index({ author: 1, status: 1 });

/**
 * 虚拟字段
 */

// 获取作者信息（需要手动 populate）
postSchema.virtual('authorInfo', {
  ref: 'User',
  localField: 'author',
  foreignField: '_id',
  justOne: true,
});

// 获取分类信息
postSchema.virtual('categoryInfo', {
  ref: 'Category',
  localField: 'category',
  foreignField: '_id',
```

```typescript
/**
 * 认证中间件
 * 验证 JWT Token 并将用户信息附加到请求对象
 */

import { Request, Response, NextFunction } from 'express';
import jwt from 'jsonwebtoken';
import { config } from '../config';
import { JWTPayload } from '../types';

// 从配置获取 JWT 密钥
const { jwt: jwtConfig } = config;

/**
 * 认证中间件
 * 
 * 工作流程：
 * 1. 从请求头获取 Token
 * 2. 验证 Token 格式
 * 3. 验证 Token 签名
 * 4. 将用户信息附加到 req.user
 */
export const authenticate = (
  req: Request,
  res: Response,
  next: NextFunction
): void | Response => {
  try {
    // 1. 获取 Token
    // 标准做法是从 Authorization 头获取，格式为 "Bearer <token>"
    const authHeader = req.headers.authorization;
    
    if (!authHeader) {
      return res.status(401).json({
        success: false,
        message: '未提供认证 Token',
      });
    }
    
    // 2. 验证格式
    // 常见的认证头格式：Bearer token123
    // 也可能是: Token token123
    const parts = authHeader.split(' ');
    
    if (parts.length !== 2 || parts[0] !== 'Bearer') {
      return res.status(401).json({
        success: false,
        message: 'Token 格式不正确',
      });
    }
    
    const token = parts[1];
    
    // 3. 验证 Token
    // jwt.verify 会检查：
    // - Token 是否被篡改（签名验证）
    // - Token 是否过期
    const decoded = jwt.verify(token, jwtConfig.secret) as JWTPayload;
    
    // 4. 将用户信息附加到请求对象
    // 后续的控制器可以通过 req.user 访问当前用户
    (req as any).user = decoded;
    
    next();
    
  } catch (error) {
    // 处理不同的错误类型
    if (error instanceof jwt.JsonWebTokenError) {
      // Token 格式错误或签名不匹配
      return res.status(401).json({
        success: false,
        message: '无效的 Token',
      });
    }
    
    if (error instanceof jwt.TokenExpiredError) {
      // Token 已过期
      return res.status(401).json({
        success: false,
        message: 'Token 已过期，请重新登录',
      });
    }
    
    // 其他未知错误
    console.error('认证中间件错误:', error);
    return res.status(500).json({
      success: false,
      message: '认证过程中发生错误',
    });
  }
};

/**
 * 可选的认证中间件
 * 如果提供了 Token 则验证，否则继续
 * 适用于需要认证但不是必须的接口
 */
export const optionalAuth = (
  req: Request,
  res: Response,
  next: NextFunction
): void => {
  const authHeader = req.headers.authorization;
  
  if (!authHeader) {
    return next();
  }
  
  try {
    const parts = authHeader.split(' ');
    
    if (parts.length === 2 && parts[0] === 'Bearer') {
      const decoded = jwt.verify(parts[1], jwtConfig.secret) as JWTPayload;
      (req as any).user = decoded;
    }
  } catch (error) {
    // 忽略错误，继续处理
    // Token 无效时当作未登录处理
  }
  
  next();
};

/**
 * 角色检查中间件工厂
 * 创建检查特定角色的中间件
 * 
 * @param allowedRoles 允许的角色列表
 */
export const requireRole = (...allowedRoles: string[]) => {
  return (req: Request, res: Response, next: NextFunction): void | Response => {
    const user = (req as any).user;
    
    if (!user) {
      return res.status(401).json({
        success: false,
        message: '请先登录',
      });
    }
    
    // 如果没有定义角色要求，或者用户角色在允许列表中
    if (allowedRoles.length === 0 || allowedRoles.includes(user.role)) {
      return next();
    }
    
    return res.status(403).json({
      success: false,
      message: '权限不足',
    });
  };
};
```

```typescript
/**

/**
 * 全局错误处理中间件
 * 必须是四个参数的函数，Express 会识别为错误处理中间件
 */
export const errorHandler = (
  err: Error | AppError,
  req: Request,
  res: Response,
  next: NextFunction
): void => {
  // 默认错误值
  let statusCode = 500;
  let message = '服务器内部错误';
  let errors: Record<string, string> = {};
  
  // 开发环境显示详细错误
  const isDevelopment = process.env.NODE_ENV !== 'production';
  
  // 处理不同类型的错误
  if (err instanceof AppError) {
    // 自定义应用错误
    statusCode = err.statusCode;
    message = err.message;
    
    if (err instanceof ValidationError) {
      errors = err.errors;
    }
  } else if (err.name === 'ValidationError') {
    // Mongoose 验证错误
    // Mongoose 在验证失败时会设置 name 为 'ValidationError'
    statusCode = 400;
    message = '数据验证失败';
    
    // 提取字段级错误信息
    const mongooseError = err as any;
    if (mongooseError.errors) {
      Object.keys(mongooseError.errors).forEach((key) => {
        errors[key] = mongooseError.errors[key].message;
      });
    }
  } else if (err.name === 'CastError') {
    // Mongoose 类型转换错误（例如无效的 ObjectId）
    statusCode = 400;
    message = '无效的资源 ID';
  } else if ((err as any).code === 11000) {
    // MongoDB 唯一索引冲突
    statusCode = 409;
    const duplicateKey = Object.keys((err as any).keyValue || {})[0];
    message = `${duplicateKey || '字段'}已存在`;
  }
  
  // 记录错误日志
  console.error('========== 错误信息 ==========');
  console.error(`路径: ${req.method} ${req.originalUrl}`);
  console.error(`状态码: ${statusCode}`);
  console.error(`消息: ${message}`);
  console.error(`错误: ${err.message}`);
  
  if (isDevelopment) {
    console.error(`堆栈: ${err.stack}`);
  }
  
  // 构建响应
  const response: any = {
    success: false,
    message,
  };
  
  // 添加字段错误（用于表单验证）
  if (Object.keys(errors).length > 0) {
    response.errors = errors;
  }
  
  // 开发环境添加更多信息
  if (isDevelopment) {
    response.error = err.message;
    response.stack = err.stack;
  }
  
  // 发送响应
  res.status(statusCode).json(response);
};

/**
 * 异步错误包装器
 * 用于包装异步路由处理器，自动捕获 Promise 错误
 */
export const asyncHandler = (
  fn: (req: Request, res: Response, next: NextFunction) => Promise<any>
) => {
  return (req: Request, res: Response, next: NextFunction) => {
    Promise.resolve(fn(req, res, next)).catch(next);
  };
};
```

```typescript
const createHeaders = (): HeadersInit => {
  const headers: HeadersInit = {

 * 统一处理 API 响应
 * 
 * @param response Fetch 响应对象
 * @returns 解析后的 JSON 数据
 * @throws API 返回错误时抛出

const handleResponse = async <T>(response: Response): Promise<T> => {
  const data: ApiResponse<T> = await response.json();
  
  // 如果请求不成功，抛出错误
  if (!response.ok || !data.success) {
    throw new ApiError(
      data.message || '请求失败',
      response.status,
      data.errors
    );
  }
  
  return data.data as T;
};

  status: number;
  errors?: Record<string, string>;
  
  constructor(message: string, status: number, errors?: Record<string, string>) {
    super(message);
    this.name = 'ApiError';
    this.status = status;
```

```typescript
  const [token, setToken] = useState<string | null>(null);
  const [isLoading, setIsLoading] = useState(true);
  
  // 初始化：从 localStorage 恢复状态
  useEffect(() => {
    const { user: storedUser, token: storedToken } = getInitialState();
    
    if (storedToken && storedUser) {
      setUser(storedUser);
      setToken(storedToken);
      
      // 验证 Token 是否仍然有效
      authApi.getCurrentUser()
        .then((freshUser) => {
          setUser(freshUser);
          localStorage.setItem('user', JSON.stringify(freshUser));
        })
        .catch(() => {
          // Token 无效，清除存储
          authApi.logout();
          setUser(null);
          setToken(null);
        })
        .finally(() => {
          setIsLoading(false);
        });
    } else {
      setIsLoading(false);
    }
  }, []);
  
  /**
   * 登录方法
   */
  const login = useCallback(async (email: string, password: string): Promise<void> => {
    const { user: loggedInUser, token: newToken } = await authApi.login({ email, password });
    
    setUser(loggedInUser);
    setToken(newToken);
  }, []);
  
  /**
   * 注册方法
   */
  const register = useCallback(async (username: string, email: string, password: string): Promise<void> => {
    const { user: newUser, token: newToken } = await authApi.register({ username, email, password });
    
    setUser(newUser);
    setToken(newToken);
  }, []);
  
  /**
   * 登出方法
   */
  const logout = useCallback((): void => {
    authApi.logout();
    setUser(null);
    setToken(null);
  }, []);
  
  /**
   * 上下文值
   */
  const value: AuthContextType = {
    user,
    token,
    isAuthenticated: !!user && !!token,
    isLoading,
    login,
    register,
    logout,
  };
  
  return (
    <AuthContext.Provider value={value}>
      {children}
    </AuthContext.Provider>
  );
};

/**
 * 使用认证上下文的 Hook
 * 在组件中调用，获取认证状态和方法
 */
export const useAuth = (): AuthContextType => {
  const context = useContext(AuthContext);
  
  if (context === undefined) {
    throw new Error('useAuth 必须在 AuthProvider 内部使用');
  }
  
  return context;
};
```

### 1. TypeScript 类型系统

#### any vs unknown vs never
```typescript
// any: 任意类型，失去类型检查
// ⚠️ 滥用 any 会让 TypeScript 变成 AnyScript
function processAny(value: any) {
  value.foo(); // 不报错，无法自动补全
}

// unknown: 未知类型，必须进行类型检查后才能使用
// ✅ 安全的做法
function processUnknown(value: unknown) {
  if (typeof value === 'string') {
    // TypeScript 知道这是 string，可以安全使用
    console.log(value.toUpperCase());
  }
}

// never: 表示永远不会返回或永远不存在的值
// 用于穷尽性检查
type Shape = Circle | Square;

function area(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':
      return Math.PI * shape.radius ** 2;
    case 'square':
      return shape.side ** 2;
    default:
      // never: 如果忘记处理某个 case，编译时会报错
      const _exhaustive: never = shape;
      throw new Error('Unknown shape');
  }
}
```

### 问题 1: TypeScript 中 any、unknown、never 的区别是什么？

**参考答案**:

```
any:
- 任意类型，关闭类型检查
- 可以赋值给任何类型，也可以接受任何值
- 滥用会失去 TypeScript 的类型安全保障

unknown:
- 未知类型，比 any 更安全
- 任何值都可以赋值给 unknown
- 但 unknown 不能赋值给其他具体类型
- 使用前必须进行类型检查

never:
- 表示永远不存在的值
- 用于无法到达的代码（如函数总是抛出异常）
- 用于穷尽性检查（switch 的 default）
```

**答题思路**: 先解释基本概念，再举实际例子，最后说明最佳实践。

### 问题 3: Express 中间件是如何工作的？

**参考答案**:

```
Express 中间件是一个函数，签名如下：
(req, res, next) => void

工作流程：
1. 按定义顺序执行（中间件栈）
2. 每个中间件可以：
   - 处理请求
   - 修改变 req/res 对象
   - 结束响应
   - 调用 next() 传递给下一个中间件

常见中间件类型：
1. 应用级：app.use()
2. 路由级：router.use()
3. 错误处理（有4个参数）
4. 内置：express.json(), express.static()
5. 第三方：cors, helmet, morgan

执行顺序示例：
请求 → CORS → Helmet → Logger → 路由 → 控制器 → 响应
                        ↓
                    [如果出错]
                        ↓
                    错误处理中间件 → 响应
```

### 问题 4: MongoDB Mongoose 如何实现关联查询？

**参考答案**:

```typescript
// 1. 定义 Schema 时指定 ref
const postSchema = new Schema({
  author: {
    type: Schema.Types.ObjectId,
    ref: 'User'  // 关联 User 模型

// 2. 使用 populate() 填充关联数据

const post = await Post.findById(id).populate('comments');

// 5. 条件 populate
Post.findById(id).populate({
  path: 'comments',
  match: { status: 'published' },  // 只填充满足条件的文档
  select: 'content -_id',
  options: { limit: 5 }
});
```

### 问题 5: React 中 useEffect 的依赖数组你真的理解吗？

**参考答案**:

```typescript
// 1. 不提供依赖数组：每次渲染后都执行
useEffect(() => {
  console.log('每次渲染都执行');
});

// 2. 空数组：只在挂载时执行一次（类似 componentDidMount）
useEffect(() => {
  fetchData();
  return () => {
    // 清理函数（类似 componentWillUnmount）
    cancelRequest();
  };
}, []);

// 3. 有依赖：依赖变化时执行
useEffect(() => {
  fetchUser(userId);
}, [userId]);

// 4. 常见陷阱：引用类型导致无限循环
const [user, setUser] = useState(null);

useEffect(() => {
  fetchUser().then(setUser);
}, [user]); // ❌ user 是新对象，每次都是新引用 → 无限循环

// 正确做法：使用 user.id
useEffect(() => {
  fetchUser().then(setUser);
}, [user?.id]);

// 5. useCallback/useMemo 避免不必要的依赖
const handleClick = useCallback(() => {
  doSomething(id);
}, [id]);

useEffect(() => {
  handleClick();
}, [handleClick]); // ✅ 依赖稳定函数
```

**答题思路**: 结合 React 渲染机制解释依赖数组的作用，说明常见错误和最佳实践。
