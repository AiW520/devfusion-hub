# JPA深度集成 - 企业通讯录系统

## 一、项目介绍

### 1.1 项目概述
企业通讯录系统是一个基于 Spring Data JPA + PostgreSQL + Flyway 的完整企业级应用，用于管理公司内部的员工联系方式、部门架构和组织关系。

### 1.2 解决的问题
- **传统通讯录痛点**：Excel管理混乱、数据不共享、更新不及时
- **权限管理困难**：无法区分查看、编辑权限
- **数据安全**：敏感信息（手机号、邮箱）缺乏保护
- **跨部门协作**：无法快速找到跨部门的联系人

### 1.3 核心技术亮点
- **Spring Data JPA 深度应用**：复杂查询、多表关联、动态查询
- **Flyway 数据库版本管理**：自动化数据库迁移、版本控制
- **PostgreSQL 高级特性**：JSONB字段、全文检索、索引优化
- **JPA 审计功能**：自动记录创建时间、更新时间、操作人
- **多对多复杂关联**：部门-员工、角色-权限等复杂关系
- **QueryDSL 动态查询**：灵活的查询条件组合
- **缓存机制**：二级缓存、查询缓存优化性能

### 1.4 适用场景
- 企业内部信息管理系统
- 需要快速查询和更新的数据密集型应用
- 多表关联、复杂查询的业务场景
- 需要严格数据版本控制的系统
- 学习 JPA 最佳实践的项目

#### 2.5.1 基础实体类 (BaseEntity.java)

```java
package com.example.addressbook.entity;

import jakarta.persistence.*;
import lombok.Data;
import org.springframework.data.annotation.CreatedBy;
import org.springframework.data.annotation.CreatedDate;
import org.springframework.data.annotation.LastModifiedBy;
import org.springframework.data.annotation.LastModifiedDate;
import org.springframework.data.jpa.domain.support.AuditingEntityListener;

import java.time.LocalDateTime;

/**
 * 基础实体类，包含所有实体共有的字段
 *
 * JPA 审计功能：
 * 自动记录创建时间、更新时间、创建人、更新人
 * 需要配合 @EnableJpaAuditing 使用
 */
@Data  // Lombok 生成 getter/setter/toString 等方法
@MappedSuperclass  // 表示这是映射的超类，不会单独生成表
@EntityListeners(AuditingEntityListener.class)  // 启用 JPA 审计监听器
public abstract class BaseEntity {

    /**
     * 主键 ID
     * @GeneratedValue 使用 IDENTITY 策略（自增）
     */
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    /**
     * 创建时间
     * @CreatedDate 自动记录实体创建时间
     * @Column(updatable = false) 创建后不允许修改
     */
    @CreatedDate
    @Column(name = "created_time", nullable = false, updatable = false)
    private LocalDateTime createdTime;

    /**
     * 更新时间
     * @LastModifiedDate 自动记录实体最后修改时间
     */
    @LastModifiedDate
    @Column(name = "updated_time", nullable = false)
    private LocalDateTime updatedTime;

    /**
     * 创建人 ID
     * @CreatedBy 自动记录创建人（需要实现 AuditorAware）
     */
    @CreatedBy
    @Column(name = "created_by", updatable = false)
    private Long createdBy;

    /**
     * 更新人 ID
     * @LastModifiedBy 自动记录更新人（需要实现 AuditorAware）
     */
    @LastModifiedBy
    @Column(name = "updated_by")
    private Long updatedBy;

    /**
     * 是否删除（逻辑删除标记）
     * true 表示已删除，false 表示未删除
     */
    @Column(name = "is_deleted", nullable = false)
    private Boolean deleted = false;

    /**
     * 版本号（乐观锁）
     * 每次更新自动递增，防止并发修改冲突
     */
    @Version
    @Column(name = "version")
    private Integer version;
}
```

#### 2.5.2 部门实体 (Department.java)

```java
package com.example.addressbook.entity;

import jakarta.persistence.*;
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.EqualsAndHashCode;
import lombok.NoArgsConstructor;
import org.hibernate.annotations.SQLDelete;
import org.hibernate.annotations.Where;

import java.util.HashSet;
import java.util.Set;

/**
 * 部门实体类
 *
 * 树形结构部门管理，支持多级部门
 */
@Data  // Lombok 生成 getter/setter
@NoArgsConstructor  // Lombok 生成无参构造函数
@AllArgsConstructor  // Lombok 生成全参构造函数
@EqualsAndHashCode(callSuper = true, onlyExplicitlyIncluded = true)  // Lombok 生成 equals/hashCode
@Entity  // JPA 实体注解
@Table(name = "department", indexes = {
    @Index(name = "idx_dept_name", columnList = "name"),  // 部门名称索引
    @Index(name = "idx_dept_parent", columnList = "parent_id")  // 父部门索引
})
@SQLDelete(sql = "UPDATE department SET is_deleted = true WHERE id = ?")  // 逻辑删除
@Where(clause = "is_deleted = false")  // 查询时自动过滤已删除数据
public class Department extends BaseEntity {

    /**
     * 部门名称
     * @NotBlank 验证：不能为空
     */
    @Column(name = "name", nullable = false, length = 100)
    private String name;

    /**
     * 部门编码（唯一）
     * 建议格式：DEPT001, DEPT002 等
     */
    @Column(name = "code", unique = true, nullable = false, length = 50)
    private String code;

    /**
     * 部门描述
     */
    @Column(name = "description", length = 500)
    private String description;

    /**
     * 父部门 ID
     * self-referencing 关系，表示部门树形结构
     * null 表示根部门
     */
    @ManyToOne(fetch = FetchType.LAZY)  // 懒加载，避免 N+1 查询问题
    @JoinColumn(name = "parent_id")
    private Department parent;

    /**
     * 子部门列表
     * 一对多关系，一个部门可以有多个子部门
     * mappedBy = "parent" 表示关系由 Department.parent 维护
     * cascade = CascadeType.ALL 表示级联操作（保存、删除、更新等）
     * orphanRemoval = true 表示孤儿删除（从集合中移除的实体会被删除）
     */
    @OneToMany(mappedBy = "parent", cascade = CascadeType.ALL, orphanRemoval = true)
    private Set<Department> children = new HashSet<>();

    /**
     * 部门下的员工列表
     * 一对多关系，一个部门可以有多个员工
     * cascade = CascadeType.ALL 表示级联操作
     */
    @OneToMany(mappedBy = "department", cascade = CascadeType.ALL)
    private Set<Employee> employees = new HashSet<>();

    /**
     * 部门负责人 ID
     * 指向该部门的员工 ID
     */
    @Column(name = "manager_id")
    private Long managerId;

    /**
     * 部门负责人（关系映射）
     */
    @OneToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "manager_id", insertable = false, updatable = false)
    private Employee manager;

    /**
     * 部门层级深度
     * 根部门为 0，子部门递增
     */
    @Column(name = "level", nullable = false)
    private Integer level = 0;

    /**
     * 部门排序号
     * 用于同级部门的排序
     */
    @Column(name = "sort_order")
    private Integer sortOrder = 0;

    /**
     * 添加子部门
     * 双向关系维护的重要方法
     *
     * @param department 子部门
     */
    public void addChild(Department department) {
        children.add(department);
        department.setParent(this);
        // 自动设置子部门的层级深度
        department.setLevel(this.level + 1);
    }

    /**
     * 移除子部门
     * 双向关系维护的重要方法
     *
     * @param department 子部门
     */
    public void removeChild(Department department) {
        children.remove(department);
        department.setParent(null);
    }
}
```

```java
package com.example.addressbook.entity;

import jakarta.persistence.*;
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;
import org.hibernate.annotations.JdbcTypeCode;
import org.hibernate.type.SqlTypes;

/**
 * 联系方式实体
 *
 * 使用 PostgreSQL 的 JSONB 类型存储复杂的联系信息
 * JSONB 支持：索引、查询、更新等操作
 */
@Data
@NoArgsConstructor
@AllArgsConstructor
@Entity
@Table(name = "contact_info")
public class ContactInfo extends BaseEntity {

    /**
     * 联系方式类型
     * 1-工作电话 2-手机 3-邮箱 4-微信 5-钉钉 6-其他
     */
    @Enumerated(EnumType.STRING)  // 枚举存储为字符串
    @Column(name = "contact_type", nullable = false, length = 20)
    private ContactType contactType;

    /**
     * 联系方式值（如电话号码、邮箱地址）
     */
    @Column(name = "contact_value", nullable = false, length = 100)
    private String contactValue;

    /**
     * 是否主要联系方式
     * true 表示该类型的主要联系方式
     */
    @Column(name = "is_primary", nullable = false)
    private Boolean isPrimary = false;

    /**
     * 联系方式标签
     * JSONB 格式，可以存储额外的标签信息
     * 例如：["紧急", "工作", "私人"]
     */
    @JdbcTypeCode(SqlTypes.JSON)  // Hibernate 6+ 使用 JSON 类型
    @Column(name = "tags", columnDefinition = "jsonb")
    private String tags;

    /**
     * 备注
     */
    @Column(name = "remark", length = 200)
    private String remark;

    /**
     * 所属员工 ID
     */
    @Column(name = "employee_id", nullable = false)
    private Long employeeId;

    /**
     * 联系方式类型枚举
     */
    public enum ContactType {
        WORK_PHONE("工作电话"),
        MOBILE("手机"),
        EMAIL("邮箱"),
        WECHAT("微信"),
        DINGTALK("钉钉"),
        OTHER("其他");

        private final String description;

        ContactType(String description) {
            this.description = description;
        }

        public String getDescription() {
            return description;
        }
    }
}
```

```java
package com.example.addressbook.entity;

import jakarta.persistence.*;
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.EqualsAndHashCode;
import lombok.NoArgsConstructor;
import org.hibernate.annotations.Cache;
import org.hibernate.annotations.CacheConcurrencyStrategy;
import org.hibernate.annotations.SQLDelete;
import org.hibernate.annotations.Where;

import java.time.LocalDate;
import java.util.HashSet;
import java.util.Set;

/**
 * 员工实体类
 *
 * 包含员工基本信息、部门关联、联系方式等
 */
@Data
@NoArgsConstructor
@AllArgsConstructor
@EqualsAndHashCode(callSuper = true, onlyExplicitlyIncluded = true)
@Entity
@Table(name = "employee", indexes = {
    @Index(name = "idx_emp_name", columnList = "name"),
    @Index(name = "idx_emp_dept", columnList = "department_id"),
    @Index(name = "idx_emp_status", columnList = "status")
})
@SQLDelete(sql = "UPDATE employee SET is_deleted = true WHERE id = ?")
@Where(clause = "is_deleted = false")
@Cache(usage = CacheConcurrencyStrategy.READ_WRITE)  // 启用二级缓存
public class Employee extends BaseEntity {

    /**
     * 员工工号（唯一）
     * 格式：EMP0001, EMP0002 等
     */
    @Column(name = "employee_no", unique = true, nullable = false, length = 20)
    @EqualsAndHashCode.Include  // 加入 equals/hashCode 计算
    private String employeeNo;

    /**
     * 员工姓名
     */
    @Column(name = "name", nullable = false, length = 50)
    private String name;

    /**
     * 英文名
     */
    @Column(name = "english_name", length = 50)
    private String englishName;

    /**
     * 性别
     */
    @Enumerated(EnumType.STRING)
    @Column(name = "gender", length = 10)
    private Gender gender;

    /**
     * 出生日期
     */
    @Column(name = "birth_date")
    private LocalDate birthDate;

    /**
     * 入职日期
     */
    @Column(name = "hire_date", nullable = false)
    private LocalDate hireDate;

    /**
     * 职位
     */
    @Column(name = "position", length = 50)
    private String position;

    /**
     * 职级
     * 例如：P5, P6, P7, P8 等
     */
    @Column(name = "level", length = 20)
    private String level;

    /**
     * 邮箱
     */
    @Column(name = "email", length = 100)
    private String email;

    /**
     * 所属部门
     * 多对一关系，多个员工属于一个部门
     * FetchType.LAZY 懒加载，避免 N+1 问题
     */
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "department_id", nullable = false)
    private Department department;

    /**
     * 状态
     * ACTIVE-在职  RESIGNED-离职  SUSPENDED-停职
     */
    @Enumerated(EnumType.STRING)
    @Column(name = "status", nullable = false, length = 20)
    private EmployeeStatus status = EmployeeStatus.ACTIVE;

    /**
     * 联系方式列表
     * 一对多关系，一个员工可以有多个联系方式
     * cascade = CascadeType.ALL 级联操作
     * orphanRemoval = true 孤儿删除
```

```java
import com.example.addressbook.entity.Employee;
import com.example.addressbook.repository.EmployeeCustomRepository;
import jakarta.persistence.EntityManager;
import jakarta.persistence.PersistenceContext;
import jakarta.persistence.Query;
import org.springframework.stereotype.Repository;

import java.util.List;

/**
 * 自定义员工 Repository 实现
 *
 * 使用 EntityManager 实现复杂的数据访问逻辑
 */
@Repository
public class EmployeeCustomRepositoryImpl implements EmployeeCustomRepository {

    /**
     * JPA 实体管理器
     * 用于执行原生 SQL 和 JPQL
     */
    @PersistenceContext  // 自动注入 EntityManager
    private EntityManager entityManager;

    /**
     * 批量插入员工
     *
     * @param employees 员工列表
     * @return 插入的记录数
     */
    @Override
    public int batchInsert(List<Employee> employees) {
        int count = 0;

        for (Employee employee : employees) {
            // 使用 persist 方法保存实体
            entityManager.persist(employee);

            // 每插入 20 条记录 flush 和 clear 一次
            // 避免一级缓存过大导致性能问题
            if (++count % 20 == 0) {
                entityManager.flush();
                entityManager.clear();
            }
        }

        // 最后一次 flush
        entityManager.flush();
        return count;
    }

    /**
     * 批量更新员工部门
     *
     * @param employeeIds 员工 ID 列表
     * @param newDepartmentId 新部门 ID
     * @return 更新的记录数
     */
    @Override
    public int batchUpdateDepartment(List<Long> employeeIds, Long newDepartmentId) {
        // 使用原生 SQL 批量更新
        String sql = "UPDATE employee SET department_id = :deptId WHERE id IN :ids";

        Query query = entityManager.createNativeQuery(sql);
        query.setParameter("deptId", newDepartmentId);
        query.setParameter("ids", employeeIds);

        return query.executeUpdate();
    }

    /**
     * 动态条件查询
     *
     * @param conditions 查询条件（JSON 格式）
     * @return 员工列表
     */
    @Override
    @SuppressWarnings("unchecked")
    public List<Employee> dynamicSearch(String conditions) {
        // 使用 PostgreSQL 的 JSONB 查询功能
        String sql = """
            SELECT * FROM employee e
            WHERE e.is_deleted = false
            AND (
                SELECT COUNT(*) FROM jsonb_each_text(conditions)
                WHERE e.*::text LIKE '%' || value || '%'
            ) > 0
            """;

        Query query = entityManager.createNativeQuery(sql, Employee.class);
        query.setParameter("conditions", conditions);

        return query.getResultList();
    }
```

```java
    @CacheEvict(value = "departments", allEntries = true)
    public void moveDepartment(Long departmentId, Long newParentId) {
        log.info("移动部门，ID：{}，新父ID：{}", departmentId, newParentId);

        Department department = departmentRepository.findById(departmentId)
                .orElseThrow(() -> new ResourceNotFoundException("部门不存在"));

        // 不能将自己作为父部门
        if (Objects.equals(departmentId, newParentId)) {
            throw new IllegalArgumentException("不能将自己作为父部门");
        }

        // 查找新父部门
        Department newParent = null;
        if (newParentId != null) {
            newParent = departmentRepository.findById(newParentId)
                    .orElseThrow(() -> new ResourceNotFoundException("新父部门不存在"));

            // 检查是否循环引用
            checkCircularReference(department, newParent);
        }

        // 移除旧的父子关系
        if (department.getParent() != null) {
            department.getParent().removeChild(department);
        }

        // 建立新的父子关系
        if (newParent != null) {
            newParent.addChild(department);
        } else {
            department.setParent(null);
            department.setLevel(0);
        }

        departmentRepository.save(department);
        log.info("部门移动成功");
    }

    /**
     * 检查循环引用
     *
     * @param department 当前部门
     * @param newParent 新父部门
     */
    private void checkCircularReference(Department department, Department newParent) {
        Department current = newParent;
        while (current != null) {
            if (Objects.equals(current.getId(), department.getId())) {
                throw new IllegalArgumentException("不能将部门移动到其子部门下，会导致循环引用");
            }
            current = current.getParent();
        }
    }
}
```

### 2.10 异常处理

#### 2.10.1 全局异常处理器 (GlobalExceptionHandler.java)

```java
package com.example.addressbook.exception;

import lombok.extern.slf4j.Slf4j;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.validation.FieldError;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

import java.time.LocalDateTime;
import java.util.HashMap;
import java.util.Map;

/**
 * 全局异常处理器
 *
 * @RestControllerAdvice：
 * - @ControllerAdvice + @ResponseBody
 * - 全局异常处理
 * - 统一异常响应格式
 */
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    /**
     * 处理资源未找到异常
     *
     * @param ex 异常对象
     * @return 错误响应
     */
    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleResourceNotFound(ResourceNotFoundException ex) {
        log.error("资源未找到：{}", ex.getMessage());

        ErrorResponse error = ErrorResponse.builder()
                .timestamp(LocalDateTime.now())
                .status(HttpStatus.NOT_FOUND.value())
                .error("Not Found")
                .message(ex.getMessage())
                .build();

        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(error);
    }

    /**
     * 处理参数校验异常
     *
     * @param ex 异常对象
     * @return 错误响应
     */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ErrorResponse> handleValidationException(
            MethodArgumentNotValidException ex) {
        log.error("参数校验失败");

        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getAllErrors().forEach(error -> {
            String fieldName = ((FieldError) error).getField();
            String errorMessage = error.getDefaultMessage();
            errors.put(fieldName, errorMessage);
        });

        ErrorResponse error = ErrorResponse.builder()
                .timestamp(LocalDateTime.now())
                .status(HttpStatus.BAD_REQUEST.value())
                .error("Validation Failed")
                .message("参数校验失败")
                .errors(errors)
                .build();

        return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(error);
    }

    /**
     * 处理非法参数异常
     *
     * @param ex 异常对象
     * @return 错误响应
     */
    @ExceptionHandler(IllegalArgumentException.class)
    public ResponseEntity<ErrorResponse> handleIllegalArgument(IllegalArgumentException ex) {
        log.error("非法参数：{}", ex.getMessage());

        ErrorResponse error = ErrorResponse.builder()
                .timestamp(LocalDateTime.now())
                .status(HttpStatus.BAD_REQUEST.value())
                .error("Bad Request")
                .message(ex.getMessage())
                .build();

        return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(error);
    }

    /**
     * 处理非法状态异常
     *
     * @param ex 异常对象
     * @return 错误响应
     */
    @ExceptionHandler(IllegalStateException.class)
    public ResponseEntity<ErrorResponse> handleIllegalState(IllegalStateException ex) {
        log.error("非法状态：{}", ex.getMessage());

        ErrorResponse error = ErrorResponse.builder()
                .timestamp(LocalDateTime.now())
                .status(HttpStatus.CONFLICT.value())
                .error("Conflict")
                .message(ex.getMessage())
                .build();

        return ResponseEntity.status(HttpStatus.CONFLICT).body(error);
    }

    /**
     * 处理其他未捕获的异常
     *
     * @param ex 异常对象
     * @return 错误响应
     */
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleException(Exception ex) {
        log.error("服务器内部错误", ex);

        ErrorResponse error = ErrorResponse.builder()
                .timestamp(LocalDateTime.now())
                .status(HttpStatus.INTERNAL_SERVER_ERROR.value())
                .error("Internal Server Error")
                .message("服务器内部错误")
                .build();

        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(error);
    }
}
```

## 三、代码关键点说明

### 3.1 JPA 实体设计要点

#### 3.1.1 继承 BaseEntity 实现审计功能
```java
@MappedSuperclass  // 映射超类，不会单独生成表
@EntityListeners(AuditingEntityListener.class)  // 启用审计监听
public abstract class BaseEntity {
    @CreatedDate  // 自动记录创建时间
    @Column(name = "created_time", nullable = false, updatable = false)
    private LocalDateTime createdTime;

    @LastModifiedDate  // 自动记录更新时间
    @Column(name = "updated_time", nullable = false)
    private LocalDateTime updatedTime;
}
```

**关键点说明：**
- `@MappedSuperclass`：不会单独生成表，属性会被子类继承
- `@EntityListeners(AuditingEntityListener.class)`：启用 JPA 审计功能
- `@CreatedDate`、`@LastModifiedDate`：自动记录时间，不需要手动设置
- 需要在配置类上添加 `@EnableJpaAuditing` 注解

**可能踩的坑：**
- 忘记添加 `@EnableJpaAuditing` 导致审计功能不生效
- 使用 `@CreatedDate` 时没有设置 `updatable = false`，导致可以修改创建时间

#### 3.1.2 逻辑删除
```java
@SQLDelete(sql = "UPDATE department SET is_deleted = true WHERE id = ?")
@Where(clause = "is_deleted = false")
public class Department extends BaseEntity {
    private Boolean deleted = false;
}
```

**关键点说明：**
- `@SQLDelete`：自定义删除 SQL，改为逻辑删除
- `@Where`：查询时自动过滤已删除的数据
- 不会真正删除数据，可以恢复

**可能踩的坑：**
- 忘记添加 `@Where` 导致查询到已删除的数据
- 使用原生 SQL 查询时需要手动过滤 `is_deleted = false`

#### 3.1.3 双向关联维护
```java
// 父端
@OneToMany(mappedBy = "parent", cascade = CascadeType.ALL, orphanRemoval = true)
private Set<Department> children = new HashSet<>();

// 子端
@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "parent_id")
private Department parent;

// 维护方法
public void addChild(Department department) {
    children.add(department);
    department.setParent(this);
}
```

**关键点说明：**
- `mappedBy`：表示关系由子端的 `parent` 字段维护
- `cascade = CascadeType.ALL`：级联操作（保存、删除、更新等）
- `orphanRemoval = true`：孤儿删除（从集合中移除的实体会被删除）
- `fetch = FetchType.LAZY`：懒加载，避免 N+1 查询问题
- 必须提供维护方法（addChild/removeChild）来同时维护两端的关系

**可能踩的坑：**
- 忘记维护双向关系导致数据不一致
- 懒加载时在事务外访问关联对象导致 LazyInitializationException
- 在 `@ManyToMany` 中不使用 `Set` 而使用 `List` 导致性能问题

#### 3.1.4 乐观锁
```java
@Version
@Column(name = "version")
private Integer version;
```

**关键点说明：**
- `@Version`：乐观锁字段，每次更新自动递增
- 并发更新时会检查版本号，版本不匹配则抛出 OptimisticLockException
- 不需要加锁，性能更好

**可能踩的坑：**
- 实体中没有版本字段，乐观锁不生效
- 更新时没有读取最新的版本号导致覆盖其他人的修改

### 3.2 Repository 层设计要点

#### 3.2.1 方法命名查询
```java
Optional<Department> findByCode(String code);
List<Department> findByNameContaining(String name);
```

**关键点说明：**
- Spring Data JPA 会根据方法名自动生成 SQL
- 支持的关键字：`findBy`, `countBy`, `deleteBy`, `existsBy`
- 支持的条件：`Containing`, `StartingWith`, `EndingWith`, `Like`, `Between`, `In`, `GreaterThan`, `LessThan`

**可能踩的坑：**
- 方法名拼写错误导致 SQL 生成失败
- 关键字顺序错误

#### 3.2.2 @Query 注解自定义查询
```java
@Query("SELECT COUNT(e) FROM Employee e WHERE e.department.id = :departmentId")
Long countEmployeesByDepartmentId(@Param("departmentId") Long departmentId);
```

**关键点说明：**
- JPQL 操作的是实体类和属性，不是表和列
- `@Param` 注解绑定参数
- `nativeQuery = true` 使用原生 SQL

**可能踩的坑：**
- JPQL 中使用表名而不是实体名
- 忘记添加 `@Param` 注解

#### 3.2.3 避免 N+1 查询
```java
// 错误写法：会导致 N+1 查询
List<Department> departments = departmentRepository.findAll();
for (Department dept : departments) {
    List<Employee> employees = dept.getEmployees();  // 每次都会查询一次
}

// 正确写法：使用 JOIN FETCH
@Query("SELECT d FROM Department d LEFT JOIN FETCH d.employees")
List<Department> findAllWithEmployees();
```

**关键点说明：**
- N+1 查询：1 次查询主表 + N 次查询关联表
- `JOIN FETCH`：立即加载关联实体，避免额外查询
- 使用 `@EntityGraph` 也可以解决

**可能踩的坑：**
- 在循环中访问懒加载的关联对象
- 使用 `JOIN FETCH` 时没有添加分页参数导致性能问题

#### 3.2.4 批量操作
```java
@Transactional
public int batchInsert(List<Employee> employees) {
    int count = 0;
    for (Employee employee : employees) {
        entityManager.persist(employee);
        if (++count % 20 == 0) {
            entityManager.flush();
            entityManager.clear();
        }
    }
    entityManager.flush();
    return count;
}
```

**关键点说明：**
- 每插入一定数量（如 20 条）后 flush 和 clear
- 避免一级缓存过大导致内存溢出
- 在配置文件中设置 `jdbc.batch_size`

**可能踩的坑：**
- 不 flush/clear 导致内存溢出
- 批量大小设置不合理

### 3.3 Service 层设计要点

#### 3.3.1 事务管理
```java
@Transactional  // 开启事务
public Department createDepartment(Department department) {
    // ...
}

@Transactional(readOnly = true)  // 只读事务，优化性能
public Department getDepartmentById(Long id) {
    // ...
}
```

**关键点说明：**
- `@Transactional`：开启事务，默认是 `REQUIRED` 传播级别
- `readOnly = true`：只读事务，优化性能
- 事务会在方法返回时提交，异常时回滚

**可能踩的坑：**
- 只读事务中修改数据导致异常
- 私有方法上添加 `@Transactional` 不生效（需要通过代理调用）
- 事务传播级别设置不当

#### 3.3.2 缓存使用
```java
@Cacheable(value = "departments", key = "#id")
public Department getDepartmentById(Long id) {
    // ...
}

@CacheEvict(value = "departments", allEntries = true)
public Department updateDepartment(Long id, Department department) {
    // ...
}
```

**关键点说明：**
- `@Cacheable`：查询时先从缓存获取，没有则查询数据库并缓存
- `@CacheEvict`：更新/删除时清除缓存
- 需要在实体上添加 `@Cache` 注解

**可能踩的坑：**
- 忘记清除缓存导致数据不一致
- 缓存 key 冲突
- 缓存雪崩（大量缓存同时失效）

### 3.4 Flyway 数据库迁移要点

#### 3.4.1 版本命名规范
```
V1__init_schema.sql
V2__insert_sample_data.sql
V3__add_indexes.sql
```

**关键点说明：**
- `V` + 版本号 + `__` + 描述 + `.sql`
- 版本号必须递增
- 描述用下划线分隔

**可能踩的坑：**
- 版本号不递增导致 Flyway 报错
- 修改已执行的迁移脚本导致校验失败

#### 3.4.2 迁移脚本最佳实践
```sql
-- 使用 IF NOT EXISTS 避免重复创建
CREATE TABLE IF NOT EXISTS department (
    -- ...
);

-- 使用事务
BEGIN;
-- 多条 SQL 语句
COMMIT;
```

**关键点说明：**
- 每个脚本应该是幂等的（可以重复执行）
- 使用事务保证数据一致性
- 不要修改已执行的迁移脚本

**可能踩的坑：**
- 迁移脚本不是幂等的
- 修改已执行的脚本后无法回滚

### 5.1 IoC 容器和依赖注入（DI）

**面试题1：什么是 IoC 和 DI？它们有什么区别？**

**参考答案：**
- **IoC（Inversion of Control，控制反转）**：是一种设计思想，将对象的创建和管理权交给容器
- **DI（Dependency Injection，依赖注入）**：是 IoC 的一种实现方式，容器自动将依赖对象注入到目标对象中

**区别：**
- IoC 是思想，DI 是实现方式
- IoC 强调控制权的转移，DI 强调依赖的自动注入

**Spring 实现方式：**
- 构造函数注入（推荐）：保证依赖不可变，避免循环依赖
- Setter 注入：可选依赖，容易导致空指针
- 字段注入：不推荐，难以测试，无法设置 final

```java
// 构造函数注入（推荐）
@Service
@RequiredArgsConstructor
public class EmployeeService {
    private final EmployeeRepository employeeRepository;
}

// Setter 注入
@Service
public class EmployeeService {
    private EmployeeRepository employeeRepository;
    @Autowired
    public void setEmployeeRepository(EmployeeRepository employeeRepository) {
        this.employeeRepository = employeeRepository;
    }
}

// 字段注入（不推荐）
@Service
public class EmployeeService {
    @Autowired
    private EmployeeRepository employeeRepository;
}
```

### 5.2 AOP 面向切面编程

**面试题2：什么是 AOP？AOP 有哪些应用场景？**

**参考答案：**
- **AOP（Aspect-Oriented Programming，面向切面编程）**：通过预编译方式和运行期动态代理实现程序功能的统一维护
- **核心概念**：
  - 切面（Aspect）：横切关注点的模块化
  - 连接点（Join Point）：程序执行的某个位置（如方法调用）
  - 切入点（Pointcut）：匹配连接点的表达式
  - 通知（Advice）：在切入点执行的代码
  - 目标对象（Target）：被代理的对象
  - 代理（Proxy）：AOP 框架创建的对象

**应用场景：**
- 日志记录
- 事务管理（@Transactional）
- 权限校验
- 性能监控
- 异常处理

**示例：**
```java
@Aspect
@Component
@Slf4j
public class LoggingAspect {

    /**
     * 定义切入点：所有 Service 层方法
     */
    @Pointcut("execution(* com.example.addressbook.service..*.*(..))")
    public void serviceLayer() {}

    /**
     * 前置通知：方法执行前
     */
    @Before("serviceLayer()")
    public void beforeAdvice(JoinPoint joinPoint) {
        log.info("方法执行前：{}", joinPoint.getSignature());
    }

    /**
     * 后置通知：方法执行后
     */
    @After("serviceLayer()")
    public void afterAdvice(JoinPoint joinPoint) {
        log.info("方法执行后：{}", joinPoint.getSignature());
    }

    /**
     * 环绕通知：控制方法的执行
     */
    @Around("serviceLayer()")
    public Object aroundAdvice(ProceedingJoinPoint joinPoint) throws Throwable {
        long startTime = System.currentTimeMillis();

        try {
            Object result = joinPoint.proceed();  // 执行目标方法
            return result;
        } finally {
            long endTime = System.currentTimeMillis();
            log.info("方法执行时间：{} ms", endTime - startTime);
        }
    }
}
```

### 5.3 Spring Boot 自动配置原理

**面试题3：Spring Boot 自动配置的原理是什么？**

**参考答案：**
- **自动配置机制**：
  1. Spring Boot 启动时扫描 `classpath` 下的 `META-INF/spring.factories` 文件
  2. 加载 `EnableAutoConfiguration` 指定的自动配置类
  3. 根据 `@Conditional` 注解判断是否需要配置 Bean

- **核心注解**：
  - `@SpringBootApplication`：包含 `@EnableAutoConfiguration`
  - `@EnableAutoConfiguration`：启用自动配置
  - `@ConditionalOnClass`：类路径下存在指定类时生效
  - `@ConditionalOnMissingBean`：容器中不存在指定 Bean 时生效
  - `@ConditionalOnProperty`：配置属性满足条件时生效

**示例：**
```java
@Configuration
@ConditionalOnClass(DataSource.class)  // 类路径下存在 DataSource 类
@EnableConfigurationProperties(DataSourceProperties.class)
public class DataSourceAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean  // 容器中不存在 DataSource Bean 时创建
    public DataSource dataSource(DataSourceProperties properties) {
        // 创建 DataSource
        return properties.initializeDataSourceBuilder().build();
    }
}
```

**如何排除自动配置：**
```java
@SpringBootApplication(exclude = {DataSourceAutoConfiguration.class})
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

// 或在配置文件中
spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration
```

### 5.4 事务管理（@Transactional）

**面试题4：@Transactional 的工作原理是什么？有哪些失效场景？**

**参考答案：**
- **工作原理**：
  - Spring AOP 为标注 `@Transactional` 的方法创建代理对象
  - 方法执行前开启事务
  - 方法执行后根据结果提交或回滚事务
  - 默认只在 `RuntimeException` 和 `Error` 时回滚

- **事务传播级别**：
  - `REQUIRED`（默认）：如果存在事务则加入，否则创建新事务
  - `REQUIRES_NEW`：总是创建新事务
  - `SUPPORTS`：如果有事务则加入，否则以非事务方式执行
  - `NOT_SUPPORTED`：以非事务方式执行
  - `MANDATORY`：必须在事务中执行，否则抛出异常
  - `NEVER`：不能在事务中执行，否则抛出异常
  - `NESTED`：嵌套事务

- **失效场景**：
  1. **方法不是 public**：AOP 代理只对 public 方法生效
  2. **类内部自调用**：不经过代理，事务不生效
  3. **异常被捕获**：异常被捕获后事务不会回滚
  4. **异常类型不匹配**：默认只回滚 `RuntimeException`
  5. **数据库引擎不支持事务**：如 MySQL 的 MyISAM

**示例：**
```java
@Service
public class EmployeeService {

    // ✅ 正确：public 方法
    @Transactional
    public void createEmployee(Employee employee) {
        // ...
    }

    // ❌ 错误：private 方法
    @Transactional
    private void privateMethod() {
        // ...
    }

    // ❌ 错误：类内部自调用，事务不生效
    @Transactional
    public void methodA() {
        methodB();  // 直接调用，不经过代理
    }

    @Transactional
    public void methodB() {
        // ...
    }

    // ✅ 正确：注入自己调用
    @Autowired
    private EmployeeService self;

    @Transactional
    public void methodA() {
        self.methodB();  // 经过代理，事务生效
    }

    // ❌ 错误：异常被捕获，事务不会回滚
    @Transactional
    public void createEmployee(Employee employee) {
        try {
            // 保存员工
            employeeRepository.save(employee);
        } catch (Exception e) {
            log.error("保存失败", e);
        }
    }

    // ✅ 正确：指定回滚异常
    @Transactional(rollbackFor = Exception.class)
    public void createEmployee(Employee employee) throws Exception {
        employeeRepository.save(employee);
    }
}
```

### 5.5 缓存机制

**面试题5：Spring Cache 的工作原理是什么？如何使用？**

**参考答案：**
- **工作原理**：
  - Spring AOP 为标注缓存注解的方法创建代理
  - 方法执行前先从缓存获取数据
  - 缓存不存在则执行方法并将结果存入缓存
  - 支持 Redis、Ehcache、Caffeine 等缓存实现

- **核心注解**：
  - `@Cacheable`：查询时先从缓存获取
  - `@CachePut`：更新缓存（总是执行方法）
  - `@CacheEvict`：删除缓存
  - `@Caching`：组合多个缓存注解
  - `@CacheConfig`：类级别的缓存配置

**示例：**
```java
@Service
@CacheConfig(cacheNames = "employees")  // 类级别缓存配置
public class EmployeeService {

    /**
     * 查询员工：先从缓存获取
     */
    @Cacheable(key = "#id")  // 使用参数作为缓存 key
    public Employee getEmployeeById(Long id) {
        return employeeRepository.findById(id).orElse(null);
    }

    /**
     * 创建员工：清除缓存
     */
    @CacheEvict(key = "#result.id")  // 使用返回值作为缓存 key
    public Employee createEmployee(Employee employee) {
        return employeeRepository.save(employee);
    }

    /**
     * 更新员工：更新缓存
     */
    @CachePut(key = "#id")  // 总是执行方法并更新缓存
    public Employee updateEmployee(Long id, Employee employee) {
        Employee existing = employeeRepository.findById(id).orElse(null);
        if (existing != null) {
            // 更新字段
            return employeeRepository.save(existing);
        }
        return null;
    }

    /**
     * 删除员工：清除缓存
     */
    @CacheEvict(key = "#id")
    public void deleteEmployee(Long id) {
        employeeRepository.deleteById(id);
    }

    /**
     * 批量删除：清除所有缓存
     */
    @CacheEvict(allEntries = true)
    public void deleteAllEmployees() {
        employeeRepository.deleteAll();
    }
}
```

**缓存问题：**
- **缓存穿透**：查询不存在的数据，缓存中没有，每次都查数据库
  - 解决方案：缓存空值、布隆过滤器
- **缓存击穿**：热点数据过期，大量请求同时查询数据库
  - 解决方案：互斥锁、热点数据永不过期
- **缓存雪崩**：大量缓存同时失效
  - 解决方案：随机过期时间、缓存预热

### 5.6 线程池配置

**面试题6：如何配置 Spring Boot 的线程池？**

**参考答案：**
- **Spring Boot 默认线程池**：`ThreadPoolTaskExecutor`
- **配置方式**：
  - `application.yml` 配置
  - `@Configuration` 类配置

**示例：**
```yaml
# application.yml 配置
spring:
  task:
    execution:
      pool:
        core-size: 5  # 核心线程数
        max-size: 10  # 最大线程数
        queue-capacity: 100  # 队列容量
        keep-alive: 60s  # 线程空闲时间
      thread-name-prefix: async-task-  # 线程名前缀
```

```java
@Configuration
@EnableAsync  // 启用异步执行
public class ThreadPoolConfig {

    @Bean("taskExecutor")
    public ThreadPoolTaskExecutor taskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();

        // 核心线程数：线程池创建时就创建的线程数
        executor.setCorePoolSize(5);

        // 最大线程数：线程池最多创建的线程数
        executor.setMaxPoolSize(10);

        // 队列容量：任务队列的大小
        executor.setQueueCapacity(100);

        // 线程空闲时间：超过核心线程数的线程在空闲后的存活时间
        executor.setKeepAliveSeconds(60);

        // 线程名前缀
        executor.setThreadNamePrefix("async-task-");

        // 拒绝策略：当队列满了且线程数达到最大值时的处理方式
        // CallerRunsPolicy：由调用线程执行任务
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());

        // 等待所有任务完成后再关闭线程池
        executor.setWaitForTasksToCompleteOnShutdown(true);

        // 等待时间
        executor.setAwaitTerminationSeconds(60);

        executor.initialize();
        return executor;
    }
}
```

**使用异步方法：**
```java
@Service
public class EmployeeService {

    /**
     * 异步发送邮件
     */
    @Async("taskExecutor")  // 指定线程池
    public void sendEmail(String to, String subject, String content) {
        // 发送邮件
        log.info("发送邮件给：{}", to);
    }
}
```

**线程池参数说明：**
- **核心线程数**：CPU 密集型任务 = CPU 核心数 + 1，IO 密集型任务 = CPU 核心数 * 2
- **最大线程数**：根据任务类型和系统资源调整
- **队列容量**：不宜过大，否则会导致内存溢出
- **拒绝策略**：
  - `AbortPolicy`：抛出异常（默认）
  - `CallerRunsPolicy`：由调用线程执行
  - `DiscardPolicy`：丢弃任务
  - `DiscardOldestPolicy`：丢弃队列中最老的任务

### 5.7 RESTful API 设计

**面试题7：RESTful API 的设计原则是什么？**

**参考答案：**
- **RESTful API 设计原则**：
  1. **资源导向**：URL 表示资源，使用名词而非动词
  2. **HTTP 方法语义**：
     - `GET`：查询资源（幂等、安全）
     - `POST`：创建资源（非幂等）
     - `PUT`：更新资源（幂等）
     - `PATCH`：部分更新资源（非幂等）
     - `DELETE`：删除资源（幂等）
  3. **状态码**：
     - `200 OK`：成功
     - `201 Created`：创建成功
     - `204 No Content`：成功但无返回内容
     - `400 Bad Request`：请求参数错误
     - `401 Unauthorized`：未认证
     - `403 Forbidden`：无权限
     - `404 Not Found`：资源不存在
     - `500 Internal Server Error`：服务器错误
  4. **版本控制**：`/api/v1/employees`
  5. **分页**：`?page=0&size=10`
  6. **排序**：`?sort=name,asc`
  7. **过滤**：`?status=ACTIVE`

**示例：**
```java
@RestController
@RequestMapping("/api/v1/employees")
public class EmployeeController {

    /**
     * 查询员工列表（分页）
     * GET /api/v1/employees?page=0&size=10
     */
    @GetMapping
    public ResponseEntity<Page<Employee>> getEmployees(Pageable pageable) {
        return ResponseEntity.ok(employeeService.getEmployees(pageable));
    }

    /**
     * 查询单个员工
     * GET /api/v1/employees/{id}
     */
    @GetMapping("/{id}")
    public ResponseEntity<Employee> getEmployeeById(@PathVariable Long id) {
        return ResponseEntity.ok(employeeService.getEmployeeById(id));
    }

    /**
     * 创建员工
     * POST /api/v1/employees
     */
    @PostMapping
    public ResponseEntity<Employee> createEmployee(@Valid @RequestBody Employee employee) {
        Employee created = employeeService.createEmployee(employee);
        return ResponseEntity.status(HttpStatus.CREATED).body(created);
    }

    /**
     * 更新员工
     * PUT /api/v1/employees/{id}
     */
    @PutMapping("/{id}")
    public ResponseEntity<Employee> updateEmployee(
            @PathVariable Long id,
            @Valid @RequestBody Employee employee) {
        return ResponseEntity.ok(employeeService.updateEmployee(id, employee));
    }

    /**
     * 删除员工
     * DELETE /api/v1/employees/{id}
     */
    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteEmployee(@PathVariable Long id) {
        employeeService.deleteEmployee(id);
        return ResponseEntity.noContent().build();
    }
}
```

## 七、运行指南

### 7.1 本地运行

1. **安装依赖**
   ```bash
   # 安装 PostgreSQL
   # macOS
   brew install postgresql
   brew services start postgresql

   # 创建数据库
   createdb address_book

   # 创建用户
   psql -d postgres
   CREATE USER address_book_user WITH PASSWORD 'password';
   GRANT ALL PRIVILEGES ON DATABASE address_book TO address_book_user;
   ```

2. **配置数据库**
   ```yaml
   # application.yml
   spring:
     datasource:
       url: jdbc:postgresql://localhost:5432/address_book
       username: address_book_user
       password: password
   ```

3. **运行项目**
   ```bash
   # 编译项目
   mvn clean package

   # 运行项目
   java -jar target/address-book-1.0.0.jar

   # 或使用 Maven 运行
   mvn spring-boot:run
   ```

4. **测试 API**
   ```bash
   # 查询根部门
   curl http://localhost:8080/api/departments/roots

   # 创建员工
   curl -X POST http://localhost:8080/api/employees \
     -H "Content-Type: application/json" \
     -d '{
       "employeeNo": "EMP0009",
       "name": "测试员工",
       "hireDate": "2024-01-01",
       "department": {"id": 1}
     }'
   ```

### 7.2 Docker 运行

```bash
# 启动所有服务（PostgreSQL + 应用）
docker-compose up -d

# 查看日志
docker-compose logs -f app

# 停止服务
docker-compose down

# 停止并删除数据
docker-compose down -v
```

### 7.3 测试

```bash
# 运行所有测试
mvn test

# 运行指定测试类
mvn test -Dtest=EmployeeServiceTest

# 生成测试覆盖率报告
mvn test jacoco:report
```

## 八、总结

这个企业通讯录系统项目涵盖了 JPA 深度应用的核心知识点：

1. **JPA 实体设计**：继承、关联、审计、逻辑删除、乐观锁
2. **Repository 层**：方法命名查询、@Query 注解、动态查询、批量操作
3. **Service 层**：事务管理、缓存、业务逻辑
4. **Controller 层**：RESTful API 设计、参数校验
5. **Flyway**：数据库版本管理
6. **异常处理**：全局异常处理
7. **性能优化**：避免 N+1 查询、缓存、索引优化

通过这个项目，你可以深入理解 JPA 的使用和 Spring Boot 的最佳实践，为面试和实际工作打下坚实基础。
