# Spring Cloud 微服务 - 电商订单系统

## 一、项目介绍

### 1.1 项目概述
电商订单系统是一个基于 Spring Cloud 的微服务架构项目，包含用户服务、订单服务、商品服务、支付服务等多个微服务，使用 Docker 容器化部署，支持高并发、高可用、可扩展的电商业务场景。

### 1.2 解决的问题
- **单点故障**：微服务拆分，避免系统崩溃
- **扩展困难**：独立扩容，按需扩展
- **开发效率低**：服务独立开发、部署、迭代
- **技术栈耦合**：不同服务使用最合适的技术

### 1.3 核心技术亮点
- **Spring Cloud 微服务架构**：服务注册与发现（Nacos）、负载均衡（Ribbon/OpenFeign）、网关（Gateway）
- **Docker 容器化**：Docker Compose 一键部署
- **消息队列（Kafka）**：异步解耦、流量削峰
- **Redis 缓存**：热点数据缓存，提升性能
- **分布式事务**：Seata 实现 TCC/Saga 模式
- **链路追踪**：Sleuth + Zipkin 全链路追踪

### 1.4 适用场景
- 电商平台
- 高并发互联网应用
- 微服务架构学习
- 分布式系统实践

#### 3.1.4 用户 Service (UserService.java)

```java
package com.ecommerce.user.service;

import com.ecommerce.user.entity.User;
import com.ecommerce.user.exception.UserException;
import com.ecommerce.user.repository.UserRepository;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.cache.annotation.CacheEvict;
import org.springframework.cache.annotation.Cacheable;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.time.Duration;
import java.util.List;

/**
 * 用户服务
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class UserService {

    private final UserRepository userRepository;
    private final RedisTemplate<String, Object> redisTemplate;

    private static final String USER_CACHE_KEY = "user:";

    /**
     * 获取用户详情（带缓存）
     */
    @Cacheable(value = "users", key = "#id")
    @Transactional(readOnly = true)
    public User getUserById(Long id) {
        log.debug("查询用户，ID：{}", id);
        
        // 先从缓存获取
        String cacheKey = USER_CACHE_KEY + id;
        User cachedUser = (User) redisTemplate.opsForValue().get(cacheKey);
        if (cachedUser != null) {
            log.debug("从缓存获取用户，ID：{}", id);
            return cachedUser;
        }

        // 从数据库查询
        User user = userRepository.findById(id)
                .orElseThrow(() -> new UserException("用户不存在"));

        // 放入缓存
        redisTemplate.opsForValue().set(cacheKey, user, Duration.ofMinutes(30));

        return user;
    }

    /**
     * 获取所有用户
     */
    @Transactional(readOnly = true)
    public List<User> getAllUsers() {
        return userRepository.findAll();
    }

    /**
     * 创建用户
     */
    @CacheEvict(value = "users", allEntries = true)
    @Transactional
    public User createUser(User user) {
        log.info("创建用户：{}", user.getUsername());

        // 检查用户名是否存在
        if (userRepository.existsByUsername(user.getUsername())) {
            throw new UserException("用户名已存在");
        }

        // 检查邮箱是否存在
        if (userRepository.existsByEmail(user.getEmail())) {
            throw new UserException("邮箱已被使用");
        }

        // 设置默认状态
        user.setStatus(User.Status.NORMAL);

        User saved = userRepository.save(user);
        log.info("用户创建成功，ID：{}", saved.getId());

        return saved;
    }

    /**
     * 更新用户
     */
    @CacheEvict(value = "users", key = "#id")
    @Transactional
    public User updateUser(Long id, User updates) {
        log.info("更新用户，ID：{}", id);

        User user = userRepository.findById(id)
                .orElseThrow(() -> new UserException("用户不存在"));

        // 更新字段
        if (updates.getRealName() != null) {
            user.setRealName(updates.getRealName());
        }
        if (updates.getPhone() != null) {
            user.setPhone(updates.getPhone());
        }
        if (updates.getGender() != null) {
            user.setGender(updates.getGender());
        }
        if (updates.getAvatarUrl() != null) {
            user.setAvatarUrl(updates.getAvatarUrl());
        }

        return userRepository.save(user);
    }

    /**
     * 删除用户
     */
    @CacheEvict(value = "users", key = "#id")
    @Transactional
    public void deleteUser(Long id) {
        log.info("删除用户，ID：{}", id);

        if (!userRepository.existsById(id)) {
            throw new UserException("用户不存在");
        }

        userRepository.deleteById(id);

        // 删除缓存
        redisTemplate.delete(USER_CACHE_KEY + id);

        log.info("用户删除成功");
    }
}
```

```java
package com.ecommerce.product.entity;

import jakarta.persistence.*;
import lombok.Data;
import org.hibernate.annotations.CreationTimestamp;
import org.hibernate.annotations.UpdateTimestamp;

import java.math.BigDecimal;
import java.time.LocalDateTime;

/**
 * 商品实体
 */
@Data
@Entity
@Table(name = "product")
public class Product {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 200)
    private String name;

    @Column(columnDefinition = "TEXT")
    private String description;

    /**
     * 商品价格
     */
    @Column(nullable = false, precision = 10, scale = 2)
    private BigDecimal price;

    /**
     * 库存数量
     */
    @Column(name = "stock_count", nullable = false)
    private Integer stockCount = 0;

    /**
     * 已售数量
     */
    @Column(name = "sold_count", nullable = false)
    private Integer soldCount = 0;

    /**
     * 商品图片（JSON 数组）
     */
    @Column(name = "images", columnDefinition = "TEXT")
    private String images;

    /**
     * 商品分类
     */
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "category_id")
    private Category category;

    /**
     * 商品状态
     * ON_SALE-在售  OFF_SALE-下架
     */
    @Enumerated(EnumType.STRING)
    @Column(nullable = false, length = 20)
    private Status status = Status.ON_SALE;

    @CreationTimestamp
    @Column(name = "created_at", nullable = false, updatable = false)
    private LocalDateTime createdAt;

    @UpdateTimestamp
    @Column(name = "updated_at", nullable = false)
    private LocalDateTime updatedAt;

    public enum Status {
        ON_SALE,  // 在售
        OFF_SALE  // 下架
    }
}

/**
 * 商品分类实体
 */

@Entity

    @Column(length = 200)
```

```java
        log.debug("查询商品，ID：{}", id);

        Product product = productRepository.findById(id)
                .orElseThrow(() -> new ProductException("商品不存在"));

        return product;
    }

    /**
     * 获取商品列表
     */
    @Transactional(readOnly = true)
    public List<Product> getProducts(Long categoryId, Product.Status status) {
        if (categoryId != null && status != null) {
            return productRepository.findByCategoryIdAndStatus(categoryId, status);
        } else if (categoryId != null) {
            return productRepository.findByCategoryId(categoryId);
        } else if (status != null) {
            return productRepository.findByStatus(status);
        } else {
            return productRepository.findAll();
        }
    }

    /**
     * 创建商品
     */
    @Transactional
    public Product createProduct(Product product) {
        log.info("创建商品：{}", product.getName());

        product.setStatus(Product.Status.ON_SALE);
        product.setSoldCount(0);

        Product saved = productRepository.save(product);
        log.info("商品创建成功，ID：{}", saved.getId());

        // 初始化库存到 Redis
        redisTemplate.opsForValue().set(
                PRODUCT_STOCK_KEY + saved.getId(),
                saved.getStockCount(),
                30,
                TimeUnit.MINUTES
        );

        return saved;
    }

    /**
     * 更新商品
     */
    @Transactional
    public Product updateProduct(Long id, Product updates) {
        log.info("更新商品，ID：{}", id);

        Product product = productRepository.findById(id)
                .orElseThrow(() -> new ProductException("商品不存在"));

        // 更新字段
        if (updates.getName() != null) {
            product.setName(updates.getName());
        }
        if (updates.getDescription() != null) {
            product.setDescription(updates.getDescription());
        }
        if (updates.getPrice() != null) {
            product.setPrice(updates.getPrice());
        }
        if (updates.getStockCount() != null) {
            product.setStockCount(updates.getStockCount());
            // 更新 Redis 库存
            redisTemplate.opsForValue().set(
                    PRODUCT_STOCK_KEY + id,
                    updates.getStockCount()
            );
        }
        if (updates.getImages() != null) {
            product.setImages(updates.getImages());
        }
        if (updates.getStatus() != null) {
            product.setStatus(updates.getStatus());
        }

        return productRepository.save(product);
    }

    /**
     * 更新库存（管理员补货）
     */
    @Transactional
    public void updateStock(Long id, Integer count) {
        log.info("更新库存，商品ID：{}，数量：{}", id, count);

        Product product = productRepository.findById(id)
                .orElseThrow(() -> new ProductException("商品不存在"));

        product.setStockCount(product.getStockCount() + count);
        productRepository.save(product);

        // 更新 Redis
        redisTemplate.opsForValue().set(
                PRODUCT_STOCK_KEY + id,
                product.getStockCount()
        );
    }

    /**
     * 扣减库存（订单下单时调用）
     * 使用 Redis 实现库存扣减，保证原子性
     */
    public void deductStock(Long id, Integer count) {
        log.info("扣减库存，商品ID：{}，数量：{}", id, count);

        String stockKey = PRODUCT_STOCK_KEY + id;

        // 从 Redis 扣减库存
        Long remainingStock = redisTemplate.opsForValue().decrement(stockKey, count);

        if (remainingStock == null || remainingStock < 0) {
            // 库存不足，回滚 Redis
            if (remainingStock != null) {
                redisTemplate.opsForValue().increment(stockKey, count);
            }
            throw new ProductException("库存不足");
        }

        // 异步更新数据库
        productRepository.decrementStock(id, count);

        log.info("库存扣减成功，剩余库存：{}", remainingStock);
    }

    /**
     * 获取实时库存（从 Redis）
     */
    public Integer getRealTimeStock(Long id) {
        String stockKey = PRODUCT_STOCK_KEY + id;
        Object stock = redisTemplate.opsForValue().get(stockKey);
        
        if (stock == null) {
            // 缓存未命中，从数据库获取
            Product product = getProductById(id);
            Integer stockCount = product.getStockCount();
            
            // 回填缓存
            redisTemplate.opsForValue().set(stockKey, stockCount, 30, TimeUnit.MINUTES);
            
            return stockCount;
        }

        return (Integer) stock;
    }
}
```

```java
package com.ecommerce.order.service;

import com.ecommerce.order.dto.CreateOrderRequest;
import com.ecommerce.order.entity.Order;
import com.ecommerce.order.entity.OrderItem;
import com.ecommerce.order.exception.OrderException;
import com.ecommerce.order.repository.OrderRepository;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.kafka.annotation.KafkaListener;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.math.BigDecimal;
import java.time.LocalDateTime;
import java.util.List;
import java.util.UUID;

/**
 * 订单服务
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class OrderService {

    private final OrderRepository orderRepository;
    private final KafkaTemplate<String, Object> kafkaTemplate;

    private static final String ORDER_TOPIC = "order-created";
    private static final String PAYMENT_TOPIC = "payment-success";

    /**
     * 创建订单
     */
    @Transactional
    public Order createOrder(CreateOrderRequest request) {
        log.info("创建订单，用户ID：{}", request.getUserId());

        // 生成订单编号
        String orderNo = generateOrderNo();

        // 创建订单
        Order order = new Order();
        order.setOrderNo(orderNo);
        order.setUserId(request.getUserId());
        order.setReceiverName(request.getReceiverName());
        order.setReceiverPhone(request.getReceiverPhone());
        order.setShippingAddress(request.getShippingAddress());
        order.setRemark(request.getRemark());
        order.setStatus(Order.Status.PENDING);

        // 计算订单金额
        BigDecimal totalAmount = BigDecimal.ZERO;

        // 添加订单项
        for (var itemReq : request.getItems()) {
            OrderItem item = new OrderItem();
            item.setProductId(itemReq.getProductId());
            item.setProductName(itemReq.getProductName());
            item.setPrice(itemReq.getPrice());
            item.setQuantity(itemReq.getQuantity());
            item.setTotalAmount(itemReq.getPrice().multiply(BigDecimal.valueOf(itemReq.getQuantity())));

            order.addItem(item);

            totalAmount = totalAmount.add(item.getTotalAmount());
        }

        order.setTotalAmount(totalAmount);

        // 保存订单
        Order saved = orderRepository.save(order);

        // 发送消息到 Kafka（异步扣减库存）
        sendOrderCreatedEvent(saved);

        log.info("订单创建成功，订单号：{}", orderNo);

        return saved;
    }

    /**
     * 发送订单创建事件
     */
    private void sendOrderCreatedEvent(Order order) {
        // 发送订单创建消息，触发库存扣减
        kafkaTemplate.send(ORDER_TOPIC, order.getOrderNo(), order);

        log.info("发送订单创建事件，订单号：{}", order.getOrderNo());
    }

    /**
     * 监听库存扣减结果
     */
    @KafkaListener(topics = "stock-deducted", groupId = "order-service")
    public void handleStockDeducted(Object message) {
        log.info("收到库存扣减结果：{}", message);
        // 根据结果更新订单状态
        // 这里可以实现复杂的业务逻辑
    }

    /**
     * 支付订单
     */
    @Transactional
    public void payOrder(Long orderId) {
        log.info("支付订单，订单ID：{}", orderId);

        Order order = orderRepository.findById(orderId)
                .orElseThrow(() -> new OrderException("订单不存在"));

        if (order.getStatus() != Order.Status.PENDING) {
            throw new OrderException("订单状态不允许支付");
        }

        // 更新订单状态
        order.setStatus(Order.Status.PAID);
        order.setPaidAt(LocalDateTime.now());
        orderRepository.save(order);

        // 发送支付成功消息
        kafkaTemplate.send(PAYMENT_TOPIC, order.getOrderNo(), order);

        log.info("订单支付成功，订单号：{}", order.getOrderNo());
    }

    /**
     * 取消订单
     */
    @Transactional
    public void cancelOrder(Long orderId) {
        log.info("取消订单，订单ID：{}", orderId);

        Order order = orderRepository.findById(orderId)
                .orElseThrow(() -> new OrderException("订单不存在"));

        if (order.getStatus() != Order.Status.PENDING) {
            throw new OrderException("订单状态不允许取消");
        }

        // 更新订单状态
        order.setStatus(Order.Status.CANCELLED);
        orderRepository.save(order);

        // 发送库存释放消息
        kafkaTemplate.send("stock-released", order.getOrderNo(), order);

        log.info("订单取消成功，订单号：{}", order.getOrderNo());
    }

    /**
     * 获取订单详情
     */
    @Transactional(readOnly = true)
    public Order getOrderById(Long id) {
        return orderRepository.findByIdWithItems(id)
                .orElseThrow(() -> new OrderException("订单不存在"));
    }

    /**
     * 获取用户订单列表
     */
    @Transactional(readOnly = true)
    public List<Order> getUserOrders(Long userId) {
        return orderRepository.findByUserIdOrderByCreatedAtDesc(userId);
    }

    /**
     * 生成订单编号
     */
    private String generateOrderNo() {
        // 格式：时间戳 + 随机数
        String timestamp = String.valueOf(System.currentTimeMillis());
        String random = UUID.randomUUID().toString().substring(0, 8).toUpperCase();
        return "ORD" + timestamp + random;
    }
```

## 五、面试要点

### 5.1 微服务架构

**面试题：什么是微服务架构？它有什么优缺点？**

**参考答案：**
- **微服务**：将大型应用拆分为多个小型服务，每个服务独立开发、部署
- **优点**：
  - 独立部署，易于扩展
  - 技术栈灵活
  - 容错性好
  - 开发效率高
- **缺点**：
  - 复杂度高
  - 分布式事务
  - 服务治理
  - 测试困难

### 5.2 服务注册与发现

**面试题：Nacos 是如何实现服务注册与发现的？**

**参考答案：**
1. **服务注册**：服务启动时向 Nacos Server 注册自己的地址
2. **服务发现**：消费者从 Nacos 获取提供者地址列表
3. **心跳机制**：服务定期向 Nacos 发送心跳，如果超时则标记为不健康
4. **负载均衡**：配合 Ribbon/OpenFeign 实现客户端负载均衡

### 5.3 分布式事务

**面试题：微服务架构下如何保证数据一致性？**

**参考答案：**
- **CAP 定理**：一致性、可用性、分区容错性只能同时满足两个
- **分布式事务方案**：
  1. **2PC/3PC**：强一致，但性能差
  2. **TCC**： Try-Confirm-Cancel
  3. **Saga**：编排多个本地事务
  4. **最终一致性**：不追求强一致，保证最终一致

### 5.4 消息队列

**面试题：Kafka 是如何保证消息不丢失的？**

**参考答案：**
- **生产者**：配置 acks=all，等待所有副本确认
- **Broker**：配置 replication.factor>=3，min.insync.replicas>=2
- **消费者**：手动提交 offset
- **消费者**：关闭自动提交，改为手动提交

### 5.5 缓存策略

**面试题：Redis 缓存与数据库如何保持一致？**

**参考答案：**
1. **Cache Aside**：先更新数据库，再删除缓存
2. **Read Through**：缓存不存在时，从数据库加载并写入缓存
3. **Write Through**：同时更新缓存和数据库
4. **延迟双删**：更新数据库后，延迟删除缓存
