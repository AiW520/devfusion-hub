## 📋 项目介绍

### 1.1 项目概述
本项目是一个**智能客服机器人**，能够基于企业知识库自动回答用户问题，支持多轮对话、意图识别和上下文理解。相比传统关键词匹配客服，AI客服能理解自然语言的复杂语义。

### 1.2 核心技术亮点
- **RAG检索增强生成**：结合向量数据库实现精准知识检索
- **LangChain核心组件**：Chain、Memory、Callback的使用
- **流式输出**：打字机效果的实时响应
- **意图识别**：支持寒暄、咨询、退款等多类意图
- **多轮对话记忆**：记住对话上下文

### 1.3 适用场景
- 企业智能客服
- 在线教育答疑
- 法律/医疗咨询助手
- 内部知识库问答

```python
"""
数据模型 - 定义 API 的输入输出数据结构
使用 Pydantic 进行数据验证

from pydantic import BaseModel, Field
from typing import List, Optional, Dict, Any
from datetime import datetime


class Message(BaseModel):
    """
    消息模型 - 表示一条聊天消息
    """
    role: str = Field(..., description="角色: user/assistant/system")
    content: str = Field(..., description="消息内容")
    timestamp: Optional[datetime] = Field(default_factory=datetime.now)


class ChatRequest(BaseModel):
    """
    聊天请求模型 - 用户发送的消息
    """
    # message: 必填，用户输入的消息
    message: str = Field(..., min_length=1, max_length=2000)
    
    # session_id: 会话ID，用于关联同一对话的历史
    # 如果不传则创建新会话
    session_id: Optional[str] = None
    
    # user_id: 用户ID（可选）
    user_id: Optional[str] = None
    
    # top_k: 召回的文档数量，默认3
    top_k: int = Field(default=3, ge=1, le=10)
    
    class Config:
        json_schema_extra = {
            "example": {
                "message": "我想退换货，怎么操作？",
                "session_id": "sess_12345",
                "top_k": 3
            }
        }


class ChatResponse(BaseModel):
    """
    聊天响应模型 - API 返回的数据结构
    """
    # answer: AI 回复的内容
    answer: str
    
    # session_id: 会话ID
    session_id: str
    
    # sources: 引用来源，用于可解释性
    sources: List[Dict[str, Any]] = Field(default_factory=list)
    
    # token_usage: token 使用量（面试常问）
    token_usage: Dict[str, int] = Field(default_factory=dict)
    
    # latency: 响应延迟（毫秒）
    latency_ms: int


class DocumentUploadRequest(BaseModel):
    """
    文档上传请求 - 用于上传知识库文档
    """
    file_name: str
    category: Optional[str] = "general"  # 文档分类


class Document(BaseModel):
    """
    文档模型 - 表示知识库中的一篇文档
    """
    id: str
    content: str
    metadata: Dict[str, Any] = Field(default_factory=dict)
    created_at: datetime = Field(default_factory=datetime.now)


class Session(BaseModel):

    会话模型 - 管理对话历史

    user_id: Optional[str]
    messages: List[Message] = Field(default_factory=list)
    created_at: datetime = Field(default_factory=datetime.now)
```

### 2.5 向量数据库服务 vector_store.py
```python
"""
向量数据库服务 - Weaviate 操作封装

为什么需要向量数据库？
- 传统数据库按关键词匹配，无法理解语义
- 向量数据库将文本转为数学向量，语义相似的文本向量也相似
- 可以实现"意思相近"的搜索，如搜索"苹果"也能找到"水果"
"""
from typing import List, Dict, Any, Optional
import weaviate
from weaviate import WeaviateClient
from weaviate.types import UUID
import os
import hashlib


class VectorStoreService:
    """
    向量存储服务类
    
    核心功能：
    1. 文档向量化存储
    2. 语义相似度检索
    3. 文档管理（删除、更新）
    """
    
    def __init__(self, url: str, api_key: Optional[str] = None):
        """
        初始化向量数据库连接
        
        Args:
            url: Weaviate 服务地址
            api_key: API密钥（Weaviate Cloud用）
        """
        # 创建 Weaviate 客户端
        # auth_client_credentials: 用于 Weaviate Cloud 的认证
        auth_config = None
        if api_key:
            auth_config = weaviate.AuthApiKey(api_key=api_key)
        
        # 连接配置
        self.client = weaviate.Client(
            url=url,
            auth_client_secret=auth_config,
            additional_headers={
                # 如果用 OpenAI 的 embedding，需要这个头
                # "X-OpenAI-Api-Key": os.getenv("OPENAI_API_KEY", "")
            }
        )
        
        # 知识库集合名称
        self.collection_name = "KnowledgeBase"
        
    def create_schema(self):
        """
        创建知识库的 schema（数据结构）
        
        Weaviate 是 schema-first 的数据库
        需要先定义数据结构才能存储数据
        """
        # 检查是否已存在，存在则删除
        if self.client.schema.exists(self.collection_name):
            self.client.schema.delete_class(self.collection_name)
        
        # 定义向量索引配置
        # VectorIndexType.HNSW: 一种高效的向量索引算法
        #   - HNSW = Hierarchical Navigable Small World
        #   - 特点：查询快、内存占用中等、支持高维向量
        vector_index_config = self.client.config.configure_vector_index(
            distance_metric="cosine"  # 余弦相似度，最适合文本语义匹配
        )
        
        # 创建集合（类似表）
        self.client.schema.create_class({
            "class": self.collection_name,
            "description": "智能客服知识库文档集合",
            "vectorizer": "text2vec-transformers",  # 使用本地 embedding 模型
            "vectorIndexConfig": vector_index_config,
            "properties": [
                {
                    "name": "content",           # 文档内容
                    "dataType": ["text"],
                    "description": "文档的文本内容"
                },
                {
                    "name": "source",             # 来源
                    "dataType": ["string"],
                    "description": "文档来源"
                },
                {
                    "name": "category",           # 分类
                    "dataType": ["string"],
                    "description": "文档分类"
                },
                {
                    "name": "file_name",          # 文件名
                    "dataType": ["string"],
                    "description": "原始文件名"
                }
            ],
            # 模块配置（使用 HuggingFace 的 embedding 模型）
            "moduleConfig": {
                "text2vec-transformers": {
                    "vectorizeClassName": False  # 类名不向量化
                }
            }
        })
        print(f"✅ Schema 创建成功: {self.collection_name}")
    
    def add_document(self, content: str, metadata: Dict[str, Any]) -> str:
        """
        添加文档到向量数据库
        
        Args:
            content: 文档文本内容
            metadata: 元数据（来源、分类等）
            
        Returns:
            文档ID
        """
        # 生成唯一ID：基于内容hash
        doc_id = hashlib.md5(content.encode()).hexdigest()
        
        # 添加到 Weaviate
        self.client.data_object.create(
            class_name=self.collection_name,
            uuid=doc_id,
            properties={
                "content": content,
                "source": metadata.get("source", ""),
                "category": metadata.get("category", "general"),
                "file_name": metadata.get("file_name", "")
            }
        )
        return doc_id
    
    def similarity_search(
        self, 
        query: str, 
        top_k: int = 3,
        filters: Optional[Dict] = None
    ) -> List[Dict[str, Any]]:
        """
        语义相似度搜索
        
        这是 RAG 的核心！根据用户问题找到最相关的知识库文档
        
        工作原理：
        1. 将用户问题转为向量
        2. 在向量数据库中找最近的K个向量
        3. 返回对应的原始文档
        
        Args:
            query: 用户问题
            top_k: 返回前K个最相关文档
            filters: 过滤条件（如只查某分类）
            
        Returns:
            相似文档列表，每个包含内容和元数据
        """
        # 构建查询
        query_builder = self.client.query.get(
            self.collection_name,
            properties=["content", "source", "category", "file_name"]
        ).with_limit(top_k)  # 限制返回数量
        
        # 添加过滤条件（可选）
        if filters:
            where_filter = self._build_filter(filters)
            query_builder = query_builder.with_where(where_filter)
        
        # 执行相似度搜索
        # near_text: 基于文本的向量搜索，内部自动转向量
        result = query_builder.with_near_text({
            "concepts": [query]  # 要搜索的文本
        }).do()
        
        # 解析结果
        documents = []
        if result.get("data", {}).get("Get", {}).get(self.collection_name):
            docs = result["data"]["Get"][self.collection_name]
            for doc in docs:
                documents.append({
                    "content": doc["content"],
                    "source": doc.get("source", ""),
                    "category": doc.get("category", ""),
                    "file_name": doc.get("file_name", "")
                })
        
        return documents
    
    def _build_filter(self, filters: Dict) -> Dict:
        """
        构建 Weaviate 查询过滤器
        
        Args:
            filters: {"category": "退货政策", "source": "官网"}
            
        Returns:
            Weaviate 格式的 where 条件
        """
        conditions = []
        for key, value in filters.items():
            conditions.append({
                "path": [key],
                "operator": "Equal",
                "valueText": value
            })
        
        if len(conditions) == 1:
            return conditions[0]
        else:
            return {
                "operator": "And",
                "operands": conditions
            }
    
    def delete_document(self, doc_id: str):
        """删除文档"""
        self.client.data_object.delete(doc_id)
    
    def get_stats(self) -> Dict[str, int]:
        """获取统计信息"""
        try:
            class_info = self.client.schema.get(self.collection_name)
            return {
                "document_count": class_info.get("objects", 0)
            }
        except:
            return {"document_count": 0}


# 全局实例（单例模式）
vector_store: Optional[VectorStoreService] = None


def get_vector_store() -> VectorStoreService:
    """获取向量存储服务实例"""
    global vector_store
    if vector_store is None:
        vector_store = VectorStoreService(
            url=os.getenv("WEAVIATE_URL", "http://localhost:8080"),
            api_key=os.getenv("WEAVIATE_API_KEY")
        )
    return vector_store
```

### 2.6 对话链 conversation_chain.py
```python
"""
对话链 - 封装 LangChain 的对话逻辑

核心概念：
- Chain: 把多个组件串联起来的流水线
- Memory: 对话历史记忆
- Callback: 回调函数（用于流式输出、日志等）
"""
from typing import Optional, List, Dict, Any, Callable
from langchain.chains import ConversationChain
from langchain.chat_models import ChatOpenAI
from langchain.memory import ConversationBufferMemory, ChatMessageHistory
from langchain.schema import HumanMessage, AIMessage, SystemMessage
from langchain.callbacks.streaming_stdout import StreamingStdOutCallbackHandler
from langchain.prompts import PromptTemplate
import os


class ConversationChainManager:
    """
    对话链管理器
    
    功能：
    1. 创建和管理对话链
    2. 支持流式输出
    3. 管理对话历史
    """
    
    def __init__(self, model_name: str = "gpt-3.5-turbo", temperature: float = 0.7):
        """
        初始化对话链管理器
        
        Args:
            model_name: LLM 模型名
            temperature: 生成温度，越高越有创意越低越确定
        """
        # 创建 ChatOpenAI 实例
        # streaming=True: 开启流式输出（打字机效果）
        # callback_manager: 回调管理器
        self.llm = ChatOpenAI(
            model=model_name,
            temperature=temperature,
            streaming=True,  # 关键：支持流式输出
            callback_manager=StreamingStdOutCallbackHandler(),  # 流式输出处理器
            api_key=os.getenv("OPENAI_API_KEY"),
            base_url=os.getenv("OPENAI_API_BASE", "https://api.openai.com/v1")
        )
        
        # 会话存储：session_id -> Memory
        self.memories: Dict[str, ConversationBufferMemory] = {}
        
        # 系统提示词（定义 AI 角色）
        # 面试重点：系统提示词工程！
        self.system_prompt = """你是一个专业的智能客服助手，名叫"小智"。
        
你的职责：
1. 礼貌、专业的回答用户问题
2. 如果知识库有相关信息，优先使用知识库回答
3. 如果知识库没有，明确告知用户
4. 回答简洁明了，不要太长
5. 如果不确定答案，诚实说不知道

回答格式：
- 先确认用户问题
- 然后给出具体回答
- 最后可以追问是否还有其他问题
"""
    
    def get_memory(self, session_id: str) -> ConversationBufferMemory:
        """
        获取或创建会话的记忆
        
        Args:
            session_id: 会话ID
            
        Returns:
            ConversationBufferMemory 实例
        """
        if session_id not in self.memories:
            # 创建新的记忆
            self.memories[session_id] = ConversationBufferMemory(
                return_messages=True,  # 返回消息对象而非字符串
                memory_key="history",  # 在 prompt 中引用的变量名
                output_key="output"    # 输出变量名
            )
        return self.memories[session_id]
    
    def create_chain(
        self, 
        session_id: str,
        system_message: Optional[str] = None
    ) -> ConversationChain:
        """
        创建对话链
        
        Args:
            session_id: 会话ID
            system_message: 自定义系统消息
            
        Returns:
            ConversationChain 实例
        """
        memory = self.get_memory(session_id)
        
        # 自定义提示词模板
        # {history}: 对话历史
        # {input}: 用户当前输入
        template = """{system_message}
        
当前对话历史：
{history}

用户：{input}

助手："""
        
        prompt = PromptTemplate(
            input_variables=["history", "input"],
            partial_variables={
                "system_message": system_message or self.system_prompt
            },
            template=template
        )
        
        # 创建对话链
        chain = ConversationChain(
            llm=self.llm,
            memory=memory,
            prompt=prompt,
            verbose=False  # 生产环境建议关闭
        )
        
        return chain
    
    async def chat(
        self,
        session_id: str,
        message: str,
        knowledge_context: Optional[str] = None
    ) -> Dict[str, Any]:
        """
        处理对话请求
        
        Args:
            session_id: 会话ID
            message: 用户消息
            knowledge_context: 知识库上下文（检索到的内容）
            
        Returns:
            {"answer": str, "token_usage": dict}
        """
        # 如果有知识库上下文，追加到输入
        if knowledge_context:
            enhanced_message = f"""请基于以下知识库内容回答用户问题：

知识库内容：
{knowledge_context}

用户问题：{message}
"""
        else:
            enhanced_message = message
        
        # 创建或获取对话链
        chain = self.create_chain(session_id)
        
        # 执行对话
        # invoke: 同步调用
        result = await chain.ainvoke({"input": enhanced_message})
        
        return {
            "answer": result["output"],
            "token_usage": result.get("token_usage", {})
        }
    
    def clear_history(self, session_id: str):
        """清除对话历史"""
        if session_id in self.memories:
            self.memories[session_id].clear()
    
    def get_history(self, session_id: str) -> List[Dict[str, str]]:
        """获取对话历史"""
        memory = self.get_memory(session_id)
        messages = memory.chat_memory.messages
        return [
            {"role": "human" if isinstance(m, HumanMessage) else "ai", "content": m.content}
            for m in messages
        ]


# 全局实例
conversation_manager: Optional[ConversationChainManager] = None


def get_conversation_manager() -> ConversationChainManager:
    """获取对话管理器实例"""
    global conversation_manager
    if conversation_manager is None:
        conversation_manager = ConversationChainManager()
    return conversation_manager
```

### 2.7 问答链 qa_chain.py
```python
"""
问答链 - 基于知识库的 RAG 实现

RAG = Retrieval Augmented Generation（检索增强生成）

工作流程：
1. 用户提问
2. 检索阶段：从向量数据库找到相关文档
3. 生成阶段：将文档和问题一起给 LLM 生成答案
"""
from typing import List, Dict, Any, Optional
from langchain.chains import RetrievalQA
from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain.chains.retrieval import create_retrieval_chain
from langchain.prompts import PromptTemplate
from langchain.schema import Document
import os


class QAService:
    """
    问答服务 - 实现 RAG 的核心逻辑
    """
    
    def __init__(self, vector_store, llm=None):
        """
        初始化问答服务
        
        Args:
            vector_store: 向量数据库实例
            llm: LLM 实例（可选，默认使用 ChatOpenAI）
        """
        self.vector_store = vector_store
        
        # 导入放在这里避免循环引用
        from langchain.chat_models import ChatOpenAI
        from langchain.callbacks.streaming_stdout import StreamingStdOutCallbackHandler
        
        # 创建 LLM
        self.llm = llm or ChatOpenAI(
            model=os.getenv("OPENAI_MODEL", "gpt-3.5-turbo"),
            temperature=0.3,  # 问答用低温度，保证准确性
            streaming=True,
            callback_manager=StreamingStdOutCallbackHandler(),
            api_key=os.getenv("OPENAI_API_KEY"),
            base_url=os.getenv("OPENAI_API_BASE", "https://api.openai.com/v1")
        )
        
        # 问答提示词
        # 关键点：让 LLM 只基于给定的文档回答，不要乱编
        self.qa_prompt = PromptTemplate.from_template(
            """你是一个专业的问答助手。请根据以下上下文内容回答用户问题。

【重要规则】
1. 只使用提供的上下文信息回答，不要编造信息
2. 如果上下文中没有答案，明确说"抱歉，我没有找到相关信息"
3. 回答要条理清晰，使用列表或分段格式
4. 引用相关原文

【上下文内容】
{context}

【用户问题】
{question}

【回答】"""
        )
        
        # 检索器配置
        self.retriever = None
    
    def create_retriever(self, top_k: int = 3):
        """
        创建检索器
        
        检索器负责从向量数据库中查找相关文档
        
        Args:
            top_k: 返回前K个最相关文档
        """
        # 使用向量存储创建检索器
        # search_type: 搜索类型（similarity / mmr）
        #   - similarity: 只返回最相似的
        #   - mmr (Maximum Marginal Relevance): 多样性搜索，避免重复
        # search_kwargs: 搜索参数
        self.retriever = self.vector_store.client.query.get(
            self.vector_store.collection_name,
            properties=["content", "source", "category"]
        ).with_limit(top_k).with_near_text({
            "concepts": ["query"]
        }).as_retriever(
            search_type="similarity",
            search_kwargs={"k": top_k}
        )
    
    async def answer(
        self, 
        question: str, 
        top_k: int = 3,
        session_id: Optional[str] = None
    ) -> Dict[str, Any]:
        """
        回答问题
        
        Args:
            question: 用户问题
            top_k: 检索的文档数量
            session_id: 会话ID（用于关联历史）
            
        Returns:
            {"answer": str, "sources": list, "token_usage": dict}
        """
        import time
        start_time = time.time()
        
        # 1. 检索相关文档
        docs = self.vector_store.similarity_search(
            query=question,
            top_k=top_k
        )
        
        # 2. 构建上下文
        context = self._build_context(docs)
        
        # 3. 构建 Prompt
        prompt = self.qa_prompt.format(
            context=context,
            question=question
        )
        
        # 4. 调用 LLM 生成答案
        # 面试重点：流式输出实现
        response = ""
        sources = []
        
        # 流式调用
        async for chunk in self.llm.astream(prompt):
            if hasattr(chunk, 'content'):
                response += chunk.content
        
        # 记录来源
        for doc in docs:
            sources.append({
                "content": doc.get("content", "")[:200] + "...",
                "source": doc.get("source", ""),
                "category": doc.get("category", "")
            })
        
        latency = int((time.time() - start_time) * 1000)
        
        return {
            "answer": response,
            "sources": sources,
            "latency_ms": latency,
            "docs_found": len(docs)
        }
    
    def _build_context(self, docs: List[Dict[str, Any]]) -> str:
        """
        构建上下文字符串
        
        将多个文档合并成一个上下文字符串
        
        Args:
            docs: 检索到的文档列表
            
        Returns:
            上下文字符串
        """
        if not docs:
            return "没有找到相关知识库内容。"
        
        context_parts = []
        for i, doc in enumerate(docs, 1):
            context_parts.append(
                f"【文档{i}】\n{doc.get('content', '')}\n"
                f"来源：{doc.get('source', '未知')}"
            )
        
        return "\n\n".join(context_parts)
```

```python
"""
文档处理服务 - 将各种格式的文档转为文本块

支持格式：PDF、Word、TXT、Markdown 等
"""
from typing import List, Dict, Any
import os
from pathlib import Path


class DocumentProcessor:
    """
    文档处理器
    
    功能：
    1. 读取多种格式的文档
    2. 文本分块（Chunking）
    3. 文本清洗
    """
    
    def __init__(self, chunk_size: int = 500, chunk_overlap: int = 50):
        """
        初始化文档处理器
        
        Args:
            chunk_size: 每个文本块的最大字符数
            chunk_overlap: 块之间的重叠字符数（避免丢失上下文）
        """
        self.chunk_size = chunk_size
        self.chunk_overlap = chunk_overlap
    
    def process_file(self, file_path: str) -> List[str]:
        """
        处理单个文件
        
        Args:
            file_path: 文件路径
            
        Returns:
            文本块列表
        """
        path = Path(file_path)
        suffix = path.suffix.lower()
        
        # 根据文件类型选择读取方法
        if suffix == ".pdf":
            return self._process_pdf(file_path)
        elif suffix in [".docx", ".doc"]:
            return self._process_docx(file_path)
        elif suffix in [".txt", ".md"]:
            return self._process_text(file_path)
        else:
            raise ValueError(f"不支持的文件格式: {suffix}")
    
    def _process_pdf(self, file_path: str) -> List[str]:
        """
        处理 PDF 文件
        
        使用 pypdf 库读取 PDF 内容
        """
        from pypdf import PdfReader
        
        text_parts = []
        reader = PdfReader(file_path)
        
        # 逐页读取
        for page_num, page in enumerate(reader.pages, 1):
            text = page.extract_text()
            if text:
                text_parts.append(text)
        
        # 合并所有文本
        full_text = "\n\n".join(text_parts)
        
        # 分块
        return self._chunk_text(full_text)
    
    def _process_docx(self, file_path: str) -> List[str]:
        """
        处理 Word 文档
        
        使用 python-docx 库读取
        """
        from docx import Document
        
        paragraphs = []
        doc = Document(file_path)
        
        # 读取所有段落
        for para in doc.paragraphs:
            text = para.text.strip()
            if text:
                paragraphs.append(text)
        
        # 合并并分块
        full_text = "\n\n".join(paragraphs)
        return self._chunk_text(full_text)
    
    def _process_text(self, file_path: str) -> List[str]:
        """
        处理纯文本文件
        """
        with open(file_path, "r", encoding="utf-8") as f:
            full_text = f.read()
        
        return self._chunk_text(full_text)
    
    def _chunk_text(self, text: str) -> List[str]:
        """
        文本分块
        
        使用滑动窗口将长文本切分成小块
        
        为什么要分块？
        - LLM 有上下文长度限制
        - 匹配更精准（小块更相关）
        - 节省 token（按量付费）
        
        Args:
            text: 原始文本
            
        Returns:
            文本块列表
        """
        chunks = []
        
        # 简单的滑动窗口分块
        start = 0
        text_length = len(text)
        
        while start < text_length:
            # 计算结束位置
            end = start + self.chunk_size
            
            # 如果不是最后一块，尝试在句号或换行处截断
            if end < text_length:
                # 向前寻找最近的句号或换行
                cut_point = self._find_best_cut_point(text, start, end)
                end = cut_point
            
            # 提取块
            chunk = text[start:end].strip()
            if chunk:
                chunks.append(chunk)
            
            # 移动窗口（考虑重叠）
            start = end - self.chunk_overlap
        
        return chunks
    
    def _find_best_cut_point(self, text: str, start: int, end: int) -> int:
        """
        找到最佳截断点
        
        优先在以下位置截断：
        1. 句号+空格（句子结尾）
        2. 换行符
        3. 逗号（中文顿号）
        """
        # 向前寻找
        for i in range(end, max(start, end - 100), -1):
            char = text[i - 1:i]
            # 句号（中文）、感叹号、问号
            if char in "。！？!\n":
                return i
        
        # 默认在指定位置截断
        return end
    
    def process_directory(
        self, 
        dir_path: str, 
        file_extensions: List[str] = [".pdf", ".docx", ".txt", ".md"]
    ) -> Dict[str, List[str]]:
        """
        处理整个目录
        
        Args:
            dir_path: 目录路径
            file_extensions: 要处理的文件扩展名列表
            
        Returns:
            {文件名: [文本块列表]}
        """
        results = {}
        path = Path(dir_path)
        
        for file_path in path.rglob("*"):
            if file_path.is_file() and file_path.suffix.lower() in file_extensions:
                try:
                    chunks = self.process_file(str(file_path))
                    results[str(file_path)] = chunks
                    print(f"✅ 处理完成: {file_path.name} ({len(chunks)} 块)")
                except Exception as e:
                    print(f"❌ 处理失败: {file_path.name}, 错误: {e}")
        
        return results


# 示例：批量导入知识库
def import_knowledge_base(vector_store, directory: str):
    """
    批量导入知识库文档
    
    Args:
        vector_store: 向量数据库实例
        directory: 文档目录路径
    """
    processor = DocumentProcessor(chunk_size=500, chunk_overlap=50)
    
    # 处理所有文档
    all_chunks = processor.process_directory(directory)
    
    # 逐块添加到向量数据库
    total_added = 0
    for file_path, chunks in all_chunks.items():
        file_name = Path(file_path).name
        
        for i, chunk in enumerate(chunks):
            metadata = {
                "source": f"{file_name} (第{i+1}块)",
                "category": "knowledge_base",
                "file_name": file_name
            }
            vector_store.add_document(chunk, metadata)
            total_added += 1
    
    print(f"✅ 知识库导入完成，共添加 {total_added} 个文档块")
    return total_added
```

```python
"""
聊天服务 - 核心业务逻辑

整合向量检索和对话生成
"""
from typing import Dict, Any, Optional, List
import uuid
from datetime import datetime

from ..services.vector_store import get_vector_store
from ..chains.conversation_chain import get_conversation_manager


class ChatService:
    """
    聊天服务 - 整合检索和生成
    
    核心流程：
    1. 用户提问 -> 向量检索 -> 获取相关文档
    2. 构建 Prompt（问题 + 检索结果 + 历史）
    3. LLM 生成回答
    4. 保存对话历史
    """
    
    def __init__(self):
        self.vector_store = get_vector_store()
        self.conversation_manager = get_conversation_manager()
        
        # 会话信息存储（生产环境用 Redis）
        self.sessions: Dict[str, Dict] = {}
    
    async def chat(
        self,
        session_id: str,
        message: str,
        user_id: Optional[str] = None,
        top_k: int = 3
    ) -> Dict[str, Any]:
        """
        处理聊天请求
        
        Args:
            session_id: 会话ID
            message: 用户消息
            user_id: 用户ID
            top_k: 检索文档数
            
        Returns:
            回复结果
        """
        # 1. 创建或更新会话
        if session_id not in self.sessions:
            self.sessions[session_id] = {
                "id": session_id,
                "user_id": user_id,
                "created_at": datetime.now(),
                "message_count": 0
            }
        
        session = self.sessions[session_id]
        session["message_count"] += 1
        
        # 2. 检索知识库
        relevant_docs = self.vector_store.similarity_search(
            query=message,
            top_k=top_k
        )
        
        # 3. 构建上下文
        context = self._build_context(relevant_docs)
        
        # 4. 调用对话链
        result = await self.conversation_manager.chat(
            session_id=session_id,
            message=message,
            knowledge_context=context if relevant_docs else None
        )
        
        # 5. 添加来源信息
        sources = []
        for doc in relevant_docs:
            sources.append({
                "content": doc.get("content", "")[:200] + "...",
                "source": doc.get("source", ""),
                "category": doc.get("category", "")
            })
        
        return {
            "answer": result["answer"],
            "sources": sources,
            "token_usage": result.get("token_usage", {}),
            "session_id": session_id
        }
    
    def _build_context(self, docs: List[Dict[str, Any]]) -> str:
        """
        构建知识库上下文
        """
        if not docs:
            return ""
        
        context_parts = []
        for i, doc in enumerate(docs, 1):
            context_parts.append(
                f"【相关文档{i}】\n"
                f"内容：{doc.get('content', '')}\n"
                f"来源：{doc.get('source', '未知')}"
            )
        
        return "\n\n".join(context_parts)
    
    def get_history(self, session_id: str) -> List[Dict[str, str]]:
        """获取对话历史"""
        return self.conversation_manager.get_history(session_id)
    
    def clear_history(self, session_id: str):
        """清除对话历史"""
        self.conversation_manager.clear_history(session_id)
        if session_id in self.sessions:
            del self.sessions[session_id]
```

```tsx
  const [loading, setLoading] = useState(false)
  // 当前回复（流式效果）
  const [currentReply, setCurrentReply] = useState('')
  
  // 消息列表滚动引用
  const messagesEndRef = useRef<HTMLDivElement>(null)
  
  // 自动滚动到底部
  useEffect(() => {
    messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' })
  }, [messages, currentReply])
  
  /**
   * 发送消息
   */
  const handleSend = async () => {
    if (!input.trim() || loading) return
    
    const userMessage: ChatMessage = {
      role: 'user',
      content: input.trim(),
      timestamp: new Date().toISOString(),
    }
    
    // 添加用户消息
    setMessages(prev => [...prev, userMessage])
    setInput('')
    setLoading(true)
    setCurrentReply('')
    
    try {
      // 调用 API
      const response: ChatResponse = await sendMessage({
        message: userMessage.content,
        session_id: sessionId,
        top_k: 3,
      })
      
      // 添加助手回复
      const assistantMessage: ChatMessage = {
        role: 'assistant',
        content: response.answer,
        timestamp: new Date().toISOString(),
      }
      
      setMessages(prev => [...prev, assistantMessage])
      
      // 显示来源（可选）
      if (response.sources.length > 0) {
        console.log('参考来源:', response.sources)
      }
      
    } catch (error) {
      console.error('发送失败:', error)
      const errorMessage: ChatMessage = {
        role: 'assistant',
        content: '抱歉，发生了错误，请稍后重试。',
        timestamp: new Date().toISOString(),
      }
      setMessages(prev => [...prev, errorMessage])
    } finally {
      setLoading(false)
      setCurrentReply('')
    }
  }
  
  /**
   * 处理键盘事件
   */
  const handleKeyPress = (e: React.KeyboardEvent) => {
    if (e.key === 'Enter' && !e.shiftKey) {
      e.preventDefault()
      handleSend()
    }
  }
  
  /**
   * 开始新对话
   */
  const handleNewChat = () => {
    const newSessionId = crypto.randomUUID()
    setMessages([])
    onSessionChange(newSessionId)
  }
  
  return (
    <div className="chat-window">
      {/* 头部 */}
      <div className="chat-header">
        <h3>🤖 智能客服</h3>
        <button onClick={handleNewChat}>新对话</button>
      </div>
      
      {/* 消息列表 */}
      <div className="messages">
        {messages.length === 0 && (
          <div className="welcome-message">
            <p>👋 你好！有什么我可以帮助你的吗？</p>
          </div>
        )}
        
        {messages.map((msg, index) => (
          <div key={index} className={`message ${msg.role}`}>
            <div className="message-avatar">
              {msg.role === 'user' ? '👤' : '🤖'}
            </div>
            <div className="message-content">
              <p>{msg.content}</p>
            </div>
          </div>
        ))}
        
        {/* 正在输入的回复 */}
        {loading && (
          <div className="message assistant">
            <div className="message-avatar">🤖</div>
            <div className="message-content">
              <p>{currentReply || '思考中...'}</p>
            </div>
          </div>
        )}
        
        <div ref={messagesEndRef} />
      </div>
      
      {/* 输入区域 */}
      <div className="input-area">
        <textarea
          value={input}
          onChange={(e) => setInput(e.target.value)}
          onKeyPress={handleKeyPress}
          placeholder="输入你的问题..."
          disabled={loading}
        />
        <button onClick={handleSend} disabled={loading || !input.trim()}>
          发送
        </button>
      </div>
    </div>
  )
}
```

## 🔍 代码关键点说明

### 3.1 RAG 检索增强生成

**什么是 RAG？为什么需要它？**

传统 LLM 的问题：
- 知识截止日期（不知道最新信息）
- 幻觉（一本正经地胡说八道）
- 无法访问私有数据

RAG 解决方案：
```
用户问题 → 向量检索 → 获取相关文档 → 拼接 Prompt → LLM 生成
```

**为什么向量检索比关键词匹配更好？**

| 对比项 | 关键词匹配 | 向量检索 |
|--------|-----------|----------|
| 搜索方式 | 精确匹配 | 语义相似 |
| "苹果" | 只能找到包含"苹果"的结果 | 能找到"水果"、"iPhone"等 |
| 同义词 | 需要手动维护同义词表 | 自动理解 |
| 语义理解 | ❌ | ✅ |

### 3.2 LangChain 核心组件

```
LangChain = LLM + Prompt + Memory + Chain + Tools

┌─────────────────────────────────────────────────────┐
│                    LangChain                        │
├─────────────────────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌────────┐ │
│  │   LLM   │  │ Prompt  │  │ Memory  │  │ Chain  │ │
│  │  (大脑) │  │ (模板)  │  │ (记忆)  │  │ (流程) │ │
│  └─────────┘  └─────────┘  └─────────┘  └────────┘ │
└─────────────────────────────────────────────────────┘
```

- **LLM**：大语言模型（GPT、Claude 等）
- **Prompt**：提示词模板
- **Memory**：对话历史记忆
- **Chain**：串联多个组件的流水线

### 3.3 Weaviate 向量数据库

**为什么选 Weaviate？**
- 开源免费
- 支持多种 embedding 模型
- 内置 HNSW 索引，高效检索
- 支持混合搜索（向量+关键词）

**HNSW 算法原理（面试常问）：**
```
Layer 2:    ●─────────────●
                    ↑
Layer 1:    ●─────●─────●─────●
            ↑         ↑
Layer 0:    ●─────●───●─────●
               ↑
Query:       ●(查询点)
```

HNSW 是一种分层导航小世界图：
- 上层：粗略搜索，快速定位大致区域
- 下层：精细搜索，精确定位最近邻

## 💼 面试要点

### 问题 1: 什么是 RAG？它的优势是什么？

**参考答案：**

RAG = Retrieval Augmented Generation（检索增强生成）

**核心思想：**
让 LLM 在回答问题前，先从知识库中检索相关信息，然后基于这些信息生成答案。

**优势（相比纯 LLM）：**
1. **解决知识时效性问题**：可以接入最新数据
2. **减少幻觉**：基于真实文档回答，不会乱编
3. **可控性强**：可以随时更新/删除知识库
4. **成本低**：比 Fine-tuning 便宜很多
5. **可解释性**：可以展示引用来源

**代码示例：**
```python
# 1. 检索相关文档
docs = vector_store.similarity_search("如何退货")

# 2. 构建上下文
context = "\n\n".join([d.content for d in docs])

# 3. 生成回答
prompt = f"基于以下信息回答：{context}\n\n问题：如何退货？"
answer = llm.invoke(prompt)
```

### 问题 2: 向量数据库是如何工作的？为什么比关键词匹配更好？

**参考答案：**

**向量数据库工作原理：**

1. **向量化阶段**：将文档切成小块（chunk），用 Embedding 模型转成向量
2. **索引阶段**：用 HNSW 等算法建立索引，加速检索
3. **检索阶段**：将查询转成向量，找最近的 K 个向量
4. **返回阶段**：返回对应原始文档

**为什么更好：**

| 维度 | 关键词匹配 | 向量检索 |
|------|-----------|----------|
| 搜索逻辑 | 精确匹配 | 语义相似 |
| 语义理解 | ❌ | ✅ |
| 同义词 | ❌ | ✅ |
| 上下文理解 | ❌ | ✅ |
| 示例 | 搜"手机"只能找到含"手机"的 | 搜"手机"能找到"iPhone"、"智能机" |

**面试加分：**
提到 HNSW 算法——一种分层导航小世界图，可以在 O(log n) 时间找到最近邻。

LangChain 核心组件：

### 问题 5: 如何实现流式输出？它的原理是什么？

**参考答案：**

**后端实现（FastAPI）：**
```python
from fastapi.responses import StreamingResponse

@app.post("/stream-chat")
async def stream_chat(message: str):
    async def event_generator():
        async for chunk in llm.astream(message):
            # SSE 格式：data: 内容\n\n
            yield f"data: {chunk.content}\n\n"
    
    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream"  # SSE MIME type
    )
```

**前端实现（React）：**
```typescript
const handleStream = async () => {
  const response = await fetch('/api/stream-chat', {
    method: 'POST',
    body: JSON.stringify({ message: input })
  })
  
  const reader = response.body.getReader()
  const decoder = new TextDecoder()
  
  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    
    const chunk = decoder.decode(value)
    setReply(prev => prev + chunk)
  }
}
```

**原理：**
- HTTP 长连接，不断发送数据块
- 前端实时显示，不用等完整响应
- 改善用户体验，感觉"在思考"
