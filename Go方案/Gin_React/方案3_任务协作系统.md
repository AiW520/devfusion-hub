## 📚 项目介绍

### 1.1 项目概述

这是一个团队协作任务管理系统（类似简版 Trello/Jira），支持多人协作、任务分配、看板管理、评论互动等功能。项目展示了 Go 在实时通信、WebSocket、多对多关系处理等方面的能力。

**核心功能：**
- 用户认证与团队管理
- 项目创建与管理
- 看板视图（待办/进行中/已完成）
- 任务 CRUD 与批量操作
- 任务评论与 @提及
- 任务标签与优先级
- WebSocket 实时通知
- 邀请成员加入团队

#### 2.2.1 WebSocket 中心 hub.go

```go
package websocket

import (
	"encoding/json"
	"log"
	"sync"

	"github.com/gorilla/websocket" // WebSocket 库
)

/*
WebSocket 核心概念：
- WebSocket 是一种 HTML5 协议，实现浏览器与服务器的 全双工通信
- 相比 HTTP 轮询，WebSocket 只需要一次握手，后续通信不需要重复请求
- 适合：实时聊天、实时协作、股票行情、游戏等场景

Hub（中心）设计模式：
- 维护所有客户端连接
- 处理消息广播
- 管理房间（团队/项目）
- 处理客户端注册/注销
*/

// Hub WebSocket 中心
// 负责管理所有 WebSocket 连接和消息路由
type Hub struct {
	// 客户端映射：clientID -> Client
	// 使用 map 存储，需要注意并发安全
	clients map[*Client]bool

	// 客户端ID到客户端的映射：用于按ID查找
	clientsByID map[string]*Client

	// 用户ID到客户端的映射：一个用户可能有多个连接（多设备）
	// value 是 *Client 集合
	users map[uint][]*Client

	// 入站消息通道
	// 所有客户端发送的消息都会进入这个通道
	// 由 Hub 的 run 方法处理
	Broadcast chan *Message

	// 注册通道
	// 客户端连接时，向此通道发送注册请求
	Register chan *Client

	// 注销通道
	// 客户端断开时，向此通道发送注销请求
	Unregister chan *Client

	// 互斥锁，保护共享数据
	mu sync.RWMutex

	// 回调函数：消息处理
	onMessage func(*Client, *Message)
}

// NewHub 创建新的 Hub
func NewHub(onMessage func(*Client, *Message)) *Hub {
	return &Hub{
		clients:      make(map[*Client]bool),
		clientsByID:  make(map[string]*Client),
		users:        make(map[uint][]*Client),
		Broadcast:    make(chan *Message, 256), // 带缓冲防止阻塞
		Register:     make(chan *Client),
		Unregister:   make(chan *Client),
		onMessage:    onMessage,
	}
}

// Run 启动 Hub
// 这是一个无限循环，必须在 goroutine 中运行
func (h *Hub) Run() {
	log.Println("🔌 WebSocket Hub 启动")

	for {
		select {
		// 处理新连接
		case client := <-h.Register:
			h.registerClient(client)

		// 处理断开连接
		case client := <-h.Unregister:
			h.unregisterClient(client)

		// 处理广播消息
		case message := <-h.Broadcast:
			h.handleBroadcast(message)
		}
	}
}

// registerClient 注册客户端
func (h *Hub) registerClient(client *Client) {
	h.mu.Lock()
	defer h.mu.Unlock()

	// 添加到 clients
	h.clients[client] = true
	h.clientsByID[client.ID] = client

	// 添加到 users（一个用户可能有多个连接）
	if h.users[client.UserID] == nil {
		h.users[client.UserID] = make([]*Client, 0)
	}
	h.users[client.UserID] = append(h.users[client.UserID], client)

	log.Printf("✅ 客户端注册: ID=%s, UserID=%d, 在线用户=%d",
		client.ID, client.UserID, len(h.users))

	// 广播用户上线状态（可选）
	h.broadcastToUserFriends(client.UserID, &Message{
		Type:    "user_online",
		Payload: map[string]interface{}{"userId": client.UserID},
	})
}

// unregisterClient 注销客户端
func (h *Hub) unregisterClient(client *Client) {
	h.mu.Lock()
	defer h.mu.Unlock()

	// 从 clients 移除
	if _, ok := h.clients[client]; ok {
		delete(h.clients, client)
		delete(h.clientsByID, client.ID)

		// 从 users 移除
		if clients, ok := h.users[client.UserID]; ok {
			// 过滤掉当前客户端
			filtered := make([]*Client, 0)
			for _, c := range clients {
				if c != client {
					filtered = append(filtered, c)
				}
			}
			if len(filtered) == 0 {
				delete(h.users, client.UserID)
			} else {
				h.users[client.UserID] = filtered
			}
		}

		// 关闭连接
		client.conn.Close()

		log.Printf("👋 客户端断开: ID=%s, UserID=%d, 在线用户=%d",
			client.ID, client.UserID, len(h.users))
	}
}

// handleBroadcast 处理广播消息
func (h *Hub) handleBroadcast(message *Message) {
	h.mu.RLock()
	defer h.mu.RUnlock()

	// 根据 TargetType 决定发送给谁
	switch message.TargetType {
	case "user":
		// 发送给指定用户
		if clients, ok := h.users[message.TargetID]; ok {
			for _, client := range clients {
				client.Send(message)
			}
		}

	case "team":
		// 发送给团队所有成员
		// 这里需要查询团队成员，为了简化，实际项目中可能使用缓存
		for client := range h.clients {
			// 实际项目中应该检查 client 是否属于目标团队
			// 简化：发送给所有在线用户
			client.Send(message)
		}

	case "project":
		// 发送给项目所有成员
		// 类似团队处理
		for client := range h.clients {
			client.Send(message)
		}

	case "all":
		// 广播给所有用户
		for client := range h.clients {
			client.Send(message)
		}
	}
}
```

#### 2.2.2 WebSocket 客户端 client.go

```go
package websocket

import (
	"encoding/json"
	"log"
	"time"

	"github.com/gorilla/websocket"
)

const (
	// 写入超时
	writeWait = 10 * time.Second

	// 读取消息超时
	readWait = 60 * time.Second

	// 发送消息缓冲大小
	sendBufSize = 256
)

// Client WebSocket 客户端
// 代表一个 WebSocket 连接
type Client struct {
	// Hub 引用
	hub *Hub

	// WebSocket 连接
	conn *websocket.Conn

	// 发送消息的通道
	// 发送方写入此通道，Hub 从中读取并发送
	send chan []byte

	// 客户端唯一ID（用于标识连接）
	ID string

	// 用户ID（从 JWT 获取）
	UserID uint

	// 用户名（用于显示）
	Username string

	// 当前所在的房间（团队/项目）
	Rooms []uint
}

// NewClient 创建新的客户端
func NewClient(hub *Hub, conn *websocket.Conn, userID uint, username string) *Client {
	return &Client{
		hub:      hub,
		conn:     conn,
		send:     make(chan []byte, sendBufSize),
		ID:       generateClientID(),
		UserID:   userID,
		Username: username,
		Rooms:    make([]uint, 0),
	}
}

// generateClientID 生成唯一客户端ID
func generateClientID() string {
	// 使用随机字符串，实际项目可使用 UUID
	return randomString(16)
}

// randomString 生成随机字符串
func randomString(n int) string {
	const letters = "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789"
	b := make([]byte, n)
	for i := range b {
		b[i] = letters[time.Now().UnixNano()%int64(len(letters))]
		time.Sleep(1 * time.Nanosecond)
	}
	return string(b)
}

// ReadPump 从 WebSocket 连接读取消息
// 在独立的 goroutine 中运行
// 读取客户端发送的消息，发送到 Hub
func (c *Client) ReadPump() {
	defer func() {
		// 连接关闭时，注销客户端
		c.hub.Unregister <- c
		c.conn.Close()
	}()

	// 设置读取超时
	c.conn.SetReadLimit(512 * 1024)        // 最大 512KB
	c.conn.SetReadDeadline(time.Now().Add(readWait))

	// 设置 Pong 处理函数
	// 客户端发送 Ping，服务器应回复 Pong
	// 实际使用心跳检测连接是否存活
	c.conn.SetPongHandler(func(string) error {
		c.conn.SetReadDeadline(time.Now().Add(readWait))
		return nil
	})

	for {
		// 读取消息
		_, message, err := c.conn.ReadMessage()
		if err != nil {
			// WebSocket 错误类型判断
			if websocket.IsUnexpectedCloseOfErr(err, websocket.CloseGoingAway, websocket.CloseAbnormalClosure) {
				log.Printf("❌ WebSocket 读取错误: %v", err)
			}
			break
		}

		// 解析消息
		var msg Message
		if err := json.Unmarshal(message, &msg); err != nil {
			log.Printf("❌ 消息解析错误: %v", err)
			continue
		}

		// 添加发送者信息
		msg.FromUserID = c.UserID
		msg.FromUsername = c.Username

		// 调用消息处理回调
		if c.hub.onMessage != nil {
			c.hub.onMessage(c, &msg)
		}
	}
}

// WritePump 向 WebSocket 连接写入消息
// 在独立的 goroutine 中运行
// 从 Hub 获取消息，写入 WebSocket 连接
func (c *Client) WritePump() {
	// 定时发送 Ping，保持连接活跃
	ticker := time.NewTicker(30 * time.Second)
	defer func() {
		ticker.Stop()
		c.conn.Close()
	}()

	for {
		select {
		// 从 send 通道读取消息
		case message, ok := <-c.send:
			// 设置写入超时
			c.conn.SetWriteDeadline(time.Now().Add(writeWait))

			if !ok {
				// send 通道关闭，说明 Hub 要关闭连接
				c.conn.WriteMessage(websocket.CloseMessage, []byte{})
				return
			}

			// 写入消息
			if err := c.conn.WriteMessage(websocket.TextMessage, message); err != nil {
				return
			}

		// 定时 Ping
		case <-ticker.C:
			c.conn.SetWriteDeadline(time.Now().Add(writeWait))
			if err := c.conn.WriteMessage(websocket.PingMessage, nil); err != nil {
				return
			}
		}
	}
}

// Send 发送消息到客户端
// 由 Hub 调用
func (c *Client) Send(message *Message) error {
	// 序列化消息
	data, err := json.Marshal(message)
	if err != nil {
		return err
	}

	// 发送到 send 通道（非阻塞）
	select {
	case c.send <- data:
		return nil
	default:
		// 通道满，说明客户端处理速度慢
		// 可以选择关闭连接或丢弃消息
		log.Printf("⚠️  客户端 %s 发送缓冲区满，丢弃消息", c.ID)
		return nil
	}
}

// SendJSON 发送 JSON 消息
func (c *Client) SendJSON(v interface{}) error {
	data, err := json.Marshal(v)
	if err != nil {
		return err
	}

	select {
	case c.send <- data:
		return nil
	default:
		return nil
	}
}

// JoinRoom 加入房间
func (c *Client) JoinRoom(roomID uint) {
	for _, r := range c.Rooms {
		if r == roomID {
			return // 已在房间中
		}
	}
	c.Rooms = append(c.Rooms, roomID)
}

// LeaveRoom 离开房间
func (c *Client) LeaveRoom(roomID uint) {
	rooms := make([]uint, 0)
	for _, r := range c.Rooms {
		if r != roomID {
			rooms = append(rooms, r)
		}
	}
	c.Rooms = rooms
}
```

#### 2.2.3 WebSocket 消息格式 message.go

```go
package websocket

/*
WebSocket 消息格式设计：
- Type: 消息类型，决定如何处理
- Payload: 消息内容（JSON）
- FromUserID/FromUsername: 发送者信息
- TargetType/TargetID: 目标类型和ID
- Timestamp: 时间戳

示例消息：
{
    "type": "task_created",
    "payload": {
        "taskId": 123,
        "title": "完成API设计",
        "assignee": "张三"
    },
    "fromUserId": 1,
    "fromUsername": "李四",
    "targetType": "project",
    "targetId": 456,
    "timestamp": "2024-01-15T10:30:00Z"
}
*/

import "time"

// Message WebSocket 消息
type Message struct {
	// 消息类型
	// 常用类型：
	// - task_created: 任务创建
	// - task_updated: 任务更新
	// - task_deleted: 任务删除
	// - task_assigned: 任务分配
	// - comment_added: 评论添加
	// - member_joined: 成员加入
	// - ping/pong: 心跳
	// - subscribe/unsubscribe: 订阅/取消订阅
	Type string `json:"type"`

	// 消息内容（任意结构）
	Payload interface{} `json:"payload"`

	// 发送者信息
	FromUserID   uint   `json:"fromUserId,omitempty"`
	FromUsername string `json:"fromUsername,omitempty"`

	// 目标
	TargetType string `json:"targetType,omitempty"` // user, team, project, all
	TargetID   uint   `json:"targetId,omitempty"`

	// 时间戳
	Timestamp time.Time `json:"timestamp"`
}

// NewMessage 创建新消息
func NewMessage(msgType string, payload interface{}) *Message {
	return &Message{
		Type:      msgType,
		Payload:   payload,
		Timestamp: time.Now(),
```

```go
}

// Create 创建任务

func (s *TaskService) Create(userID uint, req *CreateTaskRequest) (*model.Task, error) {

	// 检查用户是否是项目成员
	isMember, err := s.projectRepo.IsMember(req.ProjectID, userID)
	if err != nil || !isMember {
		return nil, errors.New("无权在此项目中创建任务")
	}

	// ========== 第2步：创建任务 ==========
	task := &model.Task{
		ProjectID:   req.ProjectID,
		Title:       req.Title,
		Description: req.Description,
		ColumnID:    req.ColumnID,
		CreatorID:   userID,
		AssigneeID:  req.AssigneeID,
		Priority:    req.Priority,
		Status:      model.TaskStatusTodo, // 待办状态
		Order:       0, // 可以在后续计算
	}

	// 保存到数据库
	if err := s.taskRepo.Create(task); err != nil {
		return nil, err
	}

	// ========== 第3步：WebSocket 实时通知 ==========
	// 通知所有项目成员，有新任务创建
	// 这是 WebSocket 的典型应用场景：实时数据同步

	// 获取创建者信息（用于显示"谁创建了任务"）
	creator, _ := s.taskRepo.GetCreator(userID)

	// 构建通知消息
	notifyMsg := ws.NewMessage(ws.MsgTaskCreated, ws.TaskCreatedMessage{
		TaskID:       task.ID,
		Title:        task.Title,
		ProjectID:    task.ProjectID,
		AssigneeID:   task.AssigneeID,
		AssigneeName: "", // 可从 userRepo 获取
	})

	// 通过 WebSocket 发送通知
	// 发送给项目所有在线成员
	if s.wsHub != nil {
		s.wsHub.SendToTeam(project.ID, userID, notifyMsg)
	}

	// 如果分配了人，还要单独通知被分配者
	if req.AssigneeID > 0 && req.AssigneeID != userID {
		assigneeMsg := ws.NewMessage(ws.MsgTaskAssigned, map[string]interface{}{
			"taskId":    task.ID,
			"title":     task.Title,
			"projectId": task.ProjectID,
			"projectName": project.Name,
			"assignedBy": creator.Username,
		})
		s.wsHub.SendToUser(req.AssigneeID, assigneeMsg)
	}

	return task, nil
}

	Description *string `json:"description"`

	DueDate     *string `json:"dueDate"`

// Update 更新任务
// 展示如何追踪字段变更并通知
func (s *TaskService) Update(taskID, userID uint, req *UpdateTaskRequest) (*model.Task, error) {
	// ========== 第1步：获取原任务 ==========
	task, err := s.taskRepo.FindByID(taskID)
	if err != nil || task == nil {
		return nil, errors.New("任务不存在")
	}

	// 检查权限
	isMember, _ := s.projectRepo.IsMember(task.ProjectID, userID)
	isAssignee := task.AssigneeID == userID
	if !isMember && !isAssignee {
		return nil, errors.New("无权修改此任务")
	}

	// ========== 第2步：记录变更内容 ==========
	// 用于通知其他用户具体发生了什么变化
	changes := make(map[string]interface{})

	// 检查每个字段是否变更
	if req.Title != nil && *req.Title != task.Title {
		changes["title"] = map[string]string{
			"old": task.Title,
			"new": *req.Title,
		}
		task.Title = *req.Title
	}

	if req.Description != nil && *req.Description != task.Description {
		changes["description"] = "updated"
		task.Description = *req.Description
	}

	if req.ColumnID != nil && *req.ColumnID != task.ColumnID {
		changes["columnId"] = map[string]uint{
			"old": task.ColumnID,
			"new": *req.ColumnID,
		}
		task.ColumnID = *req.ColumnID
	}

	if req.AssigneeID != nil && *req.AssigneeID != task.AssigneeID {
		oldAssignee := task.AssigneeID
		task.AssigneeID = *req.AssigneeID
		changes["assignee"] = map[string]interface{}{
			"old": oldAssignee,
			"new": *req.AssigneeID,
		}
	}

	if req.Priority != nil && *req.Priority != task.Priority {
		changes["priority"] = map[string]int{
			"old": task.Priority,
			"new": *req.Priority,
		}
		task.Priority = *req.Priority
	}

	// ========== 第3步：保存更新 ==========
	if err := s.taskRepo.Update(task); err != nil {
		return nil, err
	}

	// ========== 第4步：发送变更通知 ==========
	if len(changes) > 0 {
		notifyMsg := ws.NewMessage(ws.MsgTaskUpdated, ws.TaskUpdatedMessage{
			TaskID:  task.ID,
			Changes: changes,
		})
		s.wsHub.SendToTeam(task.ProjectID, userID, notifyMsg)
	}

	// 如果重新分配了任务，通知新的被分配者
	if req.AssigneeID != nil && *req.AssigneeID != 0 {
		assigneeMsg := ws.NewMessage(ws.MsgTaskAssigned, map[string]interface{}{
			"taskId":    task.ID,
			"title":     task.Title,
			"assignedBy": userID,
		})
		s.wsHub.SendToUser(*req.AssigneeID, assigneeMsg)
	}

	return task, nil
}

// GetByProject 获取项目所有任务（看板视图）
func (s *TaskService) GetByProject(projectID, userID uint) ([]model.Task, error) {
	// 验证用户是否是项目成员
	isMember, _ := s.projectRepo.IsMember(projectID, userID)
	if !isMember {
		return nil, errors.New("无权访问此项目")
	}

	return s.taskRepo.FindByProjectID(projectID)
}

// UpdateColumn 更新任务所在列（拖拽看板时调用）
func (s *TaskService) UpdateColumn(taskID, columnID, userID uint) error {
	task, err := s.taskRepo.FindByID(taskID)
	if err != nil || task == nil {
		return errors.New("任务不存在")
	}

	// 检查权限
	isMember, _ := s.projectRepo.IsMember(task.ProjectID, userID)
	if !isMember {
		return errors.New("无权操作")
	}

	task.ColumnID = columnID
	return s.taskRepo.Update(task)
}

// Delete 删除任务
func (s *TaskService) Delete(taskID, userID uint) error {
	task, err := s.taskRepo.FindByID(taskID)
	if err != nil || task == nil {
		return errors.New("任务不存在")
	}

	// 检查权限：只有创建者或项目所有者可以删除
	if task.CreatorID != userID {
		project, _ := s.projectRepo.FindByID(task.ProjectID)
		if project == nil || project.OwnerID != userID {
			return errors.New("无权删除此任务")
		}
	}

	return s.taskRepo.Delete(taskID)
}
```

### 3.1 WebSocket 连接管理

**为什么要用 Hub 模式？**
```
传统方式：每个连接一个 goroutine 读写
问题：无法统一管理所有连接，无法广播消息

Hub 模式：
- Hub 维护所有客户端连接
- 消息通过 channel 传递
- 支持广播、发送给特定用户、发送给特定房间
```

## 🎯 面试要点

### 问题 1：WebSocket 和 HTTP 长轮询的区别是什么？

**参考答案：**
```
WebSocket：
- 全双工通信，建立连接后服务器可以主动推送
- 只需要一次握手，后续通信无需重复请求
- 连接保持，适合实时性要求高的场景
- 需要自己处理断线重连

HTTP 长轮询：
- 客户端发送请求，服务器保持连接直到有数据或超时
- 每次请求都需要建立 HTTP 连接
- 实现简单，兼容性好
- 服务器压力较大，延迟较高
```

**追问：** 如何实现断线重连？
```go
// 客户端实现
function connect() {
    ws = new WebSocket(url);
    ws.onclose = () => {
        // 指数退避重连
        setTimeout(connect, Math.min(1000 * Math.pow(2, retryCount), 30000));
    };
}
```

### 问题 2：Go 中 channel 的底层原理是什么？

**参考答案：**
```
Channel 底层实现：
- 是一个环形队列
- 有发送和接收两个指针
- 使用 mutex + condition variable 实现阻塞
- 无缓冲 channel 发送/接收会阻塞
- 有缓冲 channel 队列满/空时阻塞

关键数据结构：
type hchan struct {
    qcount   uint           // 队列中的元素数量
    dataqsiz uint           // 环形队列大小
    buf      unsafe.Pointer // 环形队列指针
    elemsize uint16         // 元素大小
    closed   uint32         // 是否关闭
    sendx    uint           // 发送索引
    recvx    uint           // 接收索引
    recvq    waitq          // 接收等待队列（Goroutine）
    sendq    waitq          // 发送等待队列（Goroutine）
    lock     mutex          // 锁
}
```

### 问题 3：如何设计一个高可用的 WebSocket 服务？

**参考答案：**
```
1. 水平扩展：
   - 使用 Redis Pub/Sub 同步消息
   - 使用负载均衡（需要 sticky session）
   - 消息持久化到 MQ

2. 连接管理：
   - 连接数监控和告警
   - 连接超时自动断开
   - 心跳检测机制

3. 消息可靠性：
   - 消息持久化
   - 消息确认机制
   - 离线消息补发

4. 安全：
   - WSS（WebSocket Secure）
   - Token 验证
   - IP 白名单
```

### 问题 4：Gin 框架中间件的执行顺序是怎样的？

**参考答案：**
```
中间件执行顺序：
1. 按注册顺序执行
2. c.Next() 决定是否调用下一个中间件
3. 响应返回时按相反顺序执行

示例：
r.Use(m1, m2, m3)  // 注册顺序

请求流程：
m1.Start -> m2.Start -> Handler -> m2.End -> m1.End

代码实现：
func (m Middleware) func(c *Context) {
    // 前置处理
    c.Next()  // 调用后续处理器
    // 后置处理
}
```

### 问题 5：如何处理 WebSocket 连接中的并发问题？

**参考答案：**
```
主要并发场景：
1. Hub 中管理多个客户端
2. 客户端的读写操作
3. 消息广播

解决方案：
1. Hub 使用 mutex 保护共享状态
2. 客户端读写使用独立的 goroutine
3. 发送使用带缓冲的 channel，防止阻塞

代码示例：
type Client struct {
    hub    *Hub
    send   chan []byte  // 使用 channel 线程安全地传递消息
    mu     sync.Mutex   // 保护写操作
}
```
